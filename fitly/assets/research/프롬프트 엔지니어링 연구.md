
# H1 — ABA 핵심 절차와 프롬프트 엔지니어링 핵심 패턴은 구조적으로 대응된다

## 주장

응용행동분석(ABA)의 세 가지 핵심 교수 절차—**과제분석(task analysis)**·**촉구 위계(prompt hierarchy)**·**행동연쇄(behavior chaining)**—는 프롬프트 엔지니어링의 세 가지 핵심 패턴—**작업 분해**·**few-shot fading**·**에이전트 워크플로우**—와 **절차적으로 동형이다(procedurally isomorphic)**. 이 동형성은 복잡 수행을 단위로 쪼개고, 단서를 점진적으로 줄이며, 이전 출력을 다음 단계의 입력으로 연결하는 공통 논리에서 기인한다.

## 배경

연구자는 본업(차트 디자인 리드)에서 시행착오로 체득한 프롬프트 설계 방식이, 특수교육학개론 수업에서 접한 자폐 스펙트럼 학생 대상 교수 절차와 구조적으로 유사함을 인지했다. 이 인지가 본 연구의 출발점.

## 근거

### 용어 직접 차용 근거 — least-to-most prompting (Zhou et al. 2023) — 교재 밖 ✅ 볼트 정식 등재 완료 (2026-05-04)

[[P2023_Zhou_LeastToMost_프롬프팅|Zhou et al. (2023)]] *Least-to-Most Prompting Enables Complex Reasoning in Large Language Models* (ICLR 2023 / arXiv:2205.10625)는 **ABA 촉구 위계의 핵심 용어(least-to-most)를 논문 제목에 그대로 차용**한 LLM 기법 논문. 논문 명시 인용: "The term least-to-most prompting is borrowed from educational psychology (Libby et al., 2008)." — Libby et al. (2008)은 ABA 자폐 학생 대상 독립 놀이 기술 습득 연구.

쉬운 하위 문제 먼저 → 점진적으로 어려운 문제를 추가해 풀게 하는 2단계 프로토콜(분해 + 순차 해결)로 기존 Chain-of-Thought 대비 복잡 추론 성능을 크게 향상. → [[최소최대_프롬프팅]] 개념 노트 신규 작성.

| ABA 과제분석·촉구 위계 | Least-to-Most Prompting (Zhou et al. 2023) |
|---|---|
| 복잡한 기술 → 하위 단계 분해 (과제분석) | 복잡한 문제 → 하위 문제 분해 (Decomposition) |
| 단계별 순차 수행 (행동연쇄) | Sequential solving (이전 답 누적 추가) |
| 최대촉구 → 점진적 철회(fading) → 독립 수행 | 쉬운 하위 문제부터 → 점진적 복잡화 |
| 목표 행동의 독립 수행 유도 | 문제의 자율 해결(zero-shot 전이) 유도 |

**실험 결과**: SCAN length split 99.7% (CoT 16.2% 대비 / 14개 예시만), DROP 비축구 82.45% (CoT 74.77%), GSM8K ≥5 단계 45.23% (CoT 39.07%).

**H1과의 연결**: LLM 기법 연구자가 ABA 임상 어휘를 *직접 차용*해 명명 + Libby et al. (2008) ABA 원전 명시 인용. "구조적 유사성" → *용어 기원 공유*라는 더 강한 층위 격상 확인. 과제분석·촉구 위계·행동연쇄 세 ABA 절차 모두와 형식적 대응.

⚠️ **교재 밖 자료**: R002/교재밖 라벨. 분해 프롬프트의 도메인 간 일반화 불가(Limitations 섹션) — CTA 도메인별 과제분석 제약과 동일. code-davinci-002 모델 의존 → GPT-4/Claude 3+ 재현 미확인.

### ReAct (Yao et al. 2023) — 교재 밖 ✅ 볼트 정식 등재 완료 (2026-05-04)

[[P2023_Yao_ReAct_추론행동|Yao et al. (2023)]] *ReAct: Synergizing Reasoning and Acting in Language Models* (ICLR 2023 / arXiv:2210.03629)는 LLM에 **Thought(추론 트레이스)→Act(행동)→Observation(환경 피드백)** 교차 루프를 부여하는 프롬프팅 패러다임. **ABA의 3요인 유관(자극-반응-결과)과 구조 동형**.

| ABA 개념 | ReAct 대응 | 비고 |
|---------|-----------|------|
| 변별 자극 (SD) | Observation (환경 피드백) | 다음 행동을 안내하는 선행 자극 |
| 촉진 (prompt) | Thought (추론 트레이스) | 반응 안내, 외부 미노출 |
| 반응 (R) | Act (행동 생성) | 환경에 실제 영향 |
| 결과 (consequence) | Observation (피드백) | 반응 후 결과이자 다음 SD |
| 과제분석 (단위 분해) | Goal decomposition in Thought | 복잡 목표 → 하위 목표 열거 |
| 최소-최대 촉진 fading | Sparse thought 조건 | 필요 위치에만 Thought 삽입 |
| 모델링 (modeling) | Few-shot trajectory 예시 | 인간 궤적을 in-context 예시로 제공 |

**실험 결과**: ALFWorld 71% (Act 45%, BUTLER 37% 대비), WebShop SR 40% (IL+RL 28.7%), HotpotQA 환각 0% (CoT 56% 대비). PaLM-540B + GPT-3 두 모델에서 일관 재현.

**H1과의 연결**: 에이전트 워크플로우(행동연쇄) ↔ Thought-Act-Obs 루프의 **직접 구조 동형**. 관련 개념 노트 → [[ReAct_추론행동루프]].

⚠️ **교재 밖 자료**: R002/교재밖 라벨. PaLM-540B 비공개 모델 (재현성 제한). ABA 교재 직접 인용 없음.

### 인접 학문 매핑 확장 후보 (CTA 경유) — 교재 밖

[[P2021_김남정_CTA_텍스트마이닝|김남정 (2021)]] 의 인지적 과제분석(CTA) 프레임은 ABA 과제분석을 인지·교육공학 측면에서 확장. CTA의 CPP 프로토콜 8개 항목은 행동 단계만 다루는 ABA 과제분석보다 LLM의 분해 단위와 더 가까운 측면이 있음:

| ABA 과제분석 | [[CTA_인지적과제분석|CTA (CPP)]] | 프롬프트 엔지니어링 |
|---|---|---|
| 행동 단계 | 활동 단계 (AC) | 작업 분해 / CoT 단계 |
| (해당 없음) | 의사결정 단계 (DC) | 분기 조건 / if-else 프롬프트 |
| (해당 없음) | 개념적 지식 (CO) | system 메시지·도메인 컨텍스트 |
| (해당 없음) | 수행 조건 (CD) | 입력 검증·전제 조건 |
| (해당 없음) | 품질 기준 (QS) | 출력 형식 명세·검증 기준 |
| (해당 없음) | 필요 도구 (TO) | 도구 호출 (function calling) |

⚠️ **교재 밖 자료** — R002 정통 교재(이성봉 2019, Cooper) 영역 외부. CTA가 ABA 직계가 아닌 인접 학문이라 *전이 손실*이 있을 수 있고, CPP 8개 분류 자체가 미검증된 외부 프레임. 강 교수 자문 미수렴. **단정 금지, 방법론 참조용**.

이 확장이 검증되면 H1은 *ABA → CTA → 프롬프트* 매개 매핑 형태로 강화 가능.

### 정제 단계 매핑 확장 후보 (Self-Refine 경유) — 교재 밖

[[P2026_허준_StableDiffusion_프롬프트엔지니어링|허준·연규필 (2026)]] 은 *프롬프트 정제도의 단계적 상승*을 baseline → my_baseline → [[Self-Refine_프롬프트|Self-Refine]] 3단계로 정량 검증. 수량 정확도가 단계가 올라갈수록 향상(0% → 27.06%)됨을 보임. ABA 측 인접 절차와 후보 매핑:

| 본 논문 정제 단계 | ABA 인접 후보 매핑 (working) | 공통 구조 |
|---|---|---|
| baseline (`A photo of {k} {object}`) | 단순 변별 자극 제시 (SD만 노출) | 최소 단서로 반응 유도 |
| my_baseline (사람이 가시성·배치 단서 추가) | 변별 자극 정밀화 / 보조 단서 (prompt) 추가 | 사람이 외부 단서로 반응 형성 보조 |
| Self-Refine (LLM이 자기 출력을 평가·재작성) | 자기관리(self-management) / 외부 코치(LLM)에 의한 행동 형성(shaping) | 평가-피드백-수정 루프 |

⚠️ **교재 밖 자료 + 다층 반증 후보**:
- LLM은 매 호출마다 stateless이므로 ABA의 *학습 누적·시간적 연속*이 결여 → Self-Refine ↔ 행동연쇄 매핑은 표면 비유에 그칠 가능성.
- ABA 자기관리는 *외부 강화 환경 의존성을 줄이는 임상 절차*이고, Self-Refine의 목적은 *과제 정확도 향상*. 기능적 목적 차이.
- 본 논문 자체가 1-step Self-Refine만 적용. 반복적 Self-Refine은 미검증.

이 확장이 검증되면 H1은 *프롬프트의 정제도가 ABA 절차의 단계적 형성과 대응*된다는 동시 차원에서 강화 가능. 단, 위 반증 후보 정밀 검토 필요.

### 프롬프트 분해 측 정밀화 (권동현 2024) — 교재 밖

[[P2024_권동현_이미지생성AI_프롬프트요소|권동현 (2024)]] 은 이미지 생성 AI 프롬프트를 **인식 측 / 기술 측** 두 축 + 각 축 **3단계**로 분해 (→ [[프롬프트_인식기술_2축]]). H1 매핑 표의 "프롬프트 엔지니어링 패턴" 컬럼을 한국어 학술 어휘로 정밀화하는 자료:

| 권동현 1단계 (축) | 2단계 (구성) | 3단계 (세부) | ABA 직접 대응 |
|---|---|---|---|
| 인식 측 (perceptual) | 이야기 구성 | 사람·공간·환경 | (해당 없음 — 행동 분해와 다른 차원) |
| 인식 측 | 컨셉 | 이미지 스타일·색·분위기 | (해당 없음) |
| 기술 측 (technology) | 이미지 구성 | 레이아웃·카메라·조명 | 자극통제 / 변별자극 디자인 (교재 밖) |
| 기술 측 | 이미지 형식 | 텍스처·비율·퀄리티 향상 | 출력 형식 명세 (CTA의 QS와 유사) |

⚠️ **교재 밖 자료** — 권동현은 *이미지 생성 AI 한정*. ABA 정통 매핑은 행동 분해(과제분석) 한정이라 권동현 2축은 직접 대응 X. 다만 **프롬프트 분해 측이 1차원 행동 단계가 아닌 2축 × 3단계 다층 분해**라는 차원적 차이 부각.

**반증 후보**: 권동현 본인도 "지각 측면의 색감과 기술 측면의 조명을 붙이기도 한다"며 2축 분리가 실사용에서 무너짐을 인정 (p.4). H1의 "ABA = 프롬프트 분해" 등치가 도구·맥락 의존을 무시한 과대 일반화일 가능성.

### 프롬프트 분해 측 정밀화 (이명원·이현수 2026, ROSES) — 교재 밖

[[P2026_이명원_ROSES_포스터디자인_프롬프트|이명원·이현수 (2026)]] 는 [[ROSES_프롬프트_프레임워크|ROSES 5요소]](Role / Objective / Scenario / Expected Solution / Steps)에 *Visual Hierarchy* 차원을 추가한 *개선된 ROSES* 를 제안하고 5 시나리오에서 [[VLM-as-Judge]](GPT-4o)로 정량 검증. **분해 차원이 풍부할수록 산출 정확도가 올라간다는 H1 핵심 주장의 디자인 도메인 정량 증거**:

| ROSES | ABA 인접 후보 매핑 (working) |
|---|---|
| Role | 변별 자극(SD)의 *맥락 설정* / 자극 통제 사전 조건 |
| Objective | 행동 목표 (behavioral objective) |
| Scenario | 자극 일반화 환경 명시 |
| Expected Solution | 학습 성과 기준 (criterion for mastery) |
| Steps | 행동 연쇄 / 과제분석 단계 |
| **Visual Hierarchy** (Refined) | [[H2_누락된_ABA원리\|H2]]의 *자극 통제 정밀화*에 해당 (교재 밖) |

⚠️ **교재 밖 자료 + 강한 반증 후보**:
- 본 사료 §4.2: 5 시나리오 중 Case 3·5(향상률 -0.3%) → ROSES 분해의 효과가 *콘텐츠 성격에 의존*. 분해 차원 = 산출 정확도 단순 단조 관계가 깨짐.
- 본 사료 §5.2: 분해의 부작용 — '동질화된 디자인(design slop)'. *과도한 분해가 산출의 다양성을 감소*시키는 임계점 존재 가능성. → H1의 약화 가능성 (분해 효과 비단조).
- ROSES 5요소 고정은 영문 약어 편의일 가능성. ABA 측에 5요소 같은 정형 구조 없음.

**도메인 의존성 패턴 (3편 종합)**: [[P2024_권동현_이미지생성AI_프롬프트요소|권동현 2024]] 은 도구 변종(미드저니/SD/파이어플라이/달·이) 의존성, [[P2026_허준_StableDiffusion_프롬프트엔지니어링|허준·연규필 2026]] 은 객체 변종(apple/cat/chair) 의존성, 이명원·이현수 (2026) 은 콘텐츠 변종(학술/인포/축제/비즈/공익) 의존성을 각각 발견. **공통: 프롬프트 분해의 효과가 일반 원리로 환원되지 않는다.** H1의 "구조적 대응" 주장에 대한 누적 반증 후보.

### 컴퓨터교육 측 어휘 매핑 (이명숙 2023) — 교재 밖

[[P2023_이명숙_ChatGPT_프롬프트엔지니어링|이명숙 (2023)]] 은 비전공자 109명 사전 설문 + 25명 수업 적용을 통해 *프롬프트 작성 가이드라인 5원칙*을 도출. 원칙 3 ([[컴퓨팅사고_분해|컴퓨팅사고의 분해]]) 은 ABA 과제분석과 **분해 논리가 동일**한 컴퓨터교육 어휘:

| 이명숙 원칙 | 정의 | ABA·CTA 인접 후보 매핑 (working) |
|---|---|---|
| 원칙 1: 목표·요구사항 인지 | 문제 파악 + 결과 형태 이해 | 행동 목표(behavioral objective) |
| 원칙 2: 명확성 (숫자·구분자·구체 명령) | "2줄로 표현" 등 수치적 명시 | 변별 자극(SD) 정밀화 — [[H2_누락된_ABA원리\|H2]] 자극 통제 정밀화의 *수치 명시*. [[60-30-10_디자인_규칙\|60-30-10 규칙]]·이명원 ROSES Visual Hierarchy와 동일 결 |
| 원칙 3: [[컴퓨팅사고_분해\|컴퓨팅사고의 분해]] | 큰 문제 분해 + 반복 개선 | [[과제분석\|task analysis]] / CoT 분해 직접 매핑 |
| 원칙 4: 사실·정보 제공 | 참조 정보·세부 정보 명시 | 변별 자극(SD)의 *맥락 제공* |
| 원칙 5: 예외 처리 명시 | 조건·예외 명시 | 분기 자극 통제 / CTA의 *수행 조건(CD)* |

⚠️ **교재 밖 자료 + 다층 반증 후보**:
- 이명숙 5원칙은 *학회 발표논문 4쪽* 분량에서 *교사 1인의 수업 통찰*로 도출. 통계적 검증 없음. 5원칙의 일반화 가능성 미검증.
- 컴퓨팅사고는 *Wing 2006* 학술 분류 위 4요소 중 1요소. ABA 과제분석은 *임상 학습 절차*. 분류 위계가 다름 → *유사 어휘 풀*로 자리매김 적절, 직접 매핑은 비대칭.
- 본 사료의 가이드라인은 *Kanaries 블로그* 일부 인용. 학술 정통 기반 약함.

**4편 통합 (분해 체계 한국어 풀)**: 권동현 2축 + 이명원 ROSES 5요소 + 허준 정제 3단계 + 이명숙 5원칙. 각각 도구·도메인·정제도·교육 차원에서 다른 분해 단위 채택. **프롬프트 분해의 *학문 전통별 변종*이 풍부**. H1의 "구조적 대응" 주장이 어느 학문 전통의 분해 체계와 가장 정합하는지 후속 검토 필요.

### 의사결정 단계 매핑 확장 후보 (박철진 2025, 베르그손 정감 경유) — 교재 밖, 인문학 차원

[[P2025_박철진_베르그손_AI잠재공간|박철진·고병정 (2025)]] 은 베르그손 *정감(affection)* — *지각과 행동 사이의 일시적 중단·간격, 망설임·선택의 잠재성 영역* — 을 AI 잠재공간 분석에 적용. CTA의 **의사결정 단계 (DC)** 와 작용 영역이 인접:

| [[CTA_인지적과제분석\|CTA (CPP)]] | 베르그손 정감 (박철진 2025) | 공통 |
|---|---|---|
| 의사결정 단계 (DC) — 판단·선택 지점·기준 | 정감(affection) — 지각/행동 사이 망설임 영역 | *행동 직전 잠재 분기 영역* |
| if-then 분기 정형화 (외현 기록 가능) | 망설임·선택의 잠재성 (의식 내부) | 분기 가능성 명시 |
| 골드 스탠더드의 의사결정 항목 | 살아있는 내면의 질적 시간 (지속) 속에 위치 | (질적 차이) |

→ CTA의 DC 항목이 *외현 기록* 영역이라면, 베르그손 정감은 *질적 시간 의식 내부* 영역. 두 차원이 *행동 직전 잠재 분기*라는 점에서 인접 — H1 매핑이 *외현 + 내재* 양쪽으로 확장 가능.

⚠️ **교재 밖 + 강한 반증 후보**:
- CTA DC = *외현 기록·기능적 분기*. 베르그손 정감 = *지속(durée) 속 질적 시간*. **시간 차원 자체가 다름** — 박철진 §5.2의 *지속 결여 = AI의 본질적 한계* 명제와 직접 충돌.
- 박철진 §5.1 표면 유사 vs §5.2 본질적 차이 — *기능적 동형*에 그칠 가능성 강함. 강 교수 자문 가치 영역.
- AI 환각이 박철진의 *통계적 유령(probabilistic ghost)* 이라면, CTA DC와 매핑은 LLM의 의사결정 *시뮬레이션*에 한정. 실제 *판단 능력*과는 거리.
- → [[AI_잠재공간_기억철학]] hub에서 누적 검토 중.

### CoT 원전 — 과제분석 ↔ Chain-of-Thought 대응의 1차 실증 (Wei et al. 2022) — 교재 밖 ✅ 볼트 정식 등재 완료 (2026-05-04)

[[P2022_Wei_ChainOfThought_LLM추론|Wei et al. (2022)]] *Chain-of-Thought Prompting Elicits Reasoning in Large Language Models* (NeurIPS 2022 / arXiv:2201.11903)는 H1 매핑 표 첫 번째 행 — **과제분석 ↔ Chain-of-Thought 분해** — 의 실증 원전. Few-shot 예시에 중간 추론 단계를 포함시키면 LLM의 다단계 추론이 극적으로 향상됨을 GSM8K·StrategyQA 등 12개 벤치마크에서 확인. → [[chain_of_thought_프롬프팅]] concept 신규 작성.

| ABA 과제분석 | Chain-of-Thought (Wei et al. 2022) | 공통 구조 |
|---|---|---|
| 복잡 기술 → 하위 단계 분해 | 복잡 문제 → 중간 추론 단계 분해 | 암묵적 수행을 명시적 단위로 쪼갬 |
| 단계별 순차 수행 (행동연쇄) | 이전 추론 단계 → 다음 단계 입력 | 앞 출력이 뒤 입력의 변별자극 |
| "think out loud" (CTA 자기 보고) | 자연어 추론 단계 외재화 | 내적 처리의 언어적 외재화 |

**핵심 발견이 H1에 주는 함의**:
- CoT 효과는 **~100B 파라미터 이상 LLM에서만 창발** — ABA 측 "선행 기술 보유 전제"와 유사. LLM 보조 도구 선택 하한 기준.
- Standard prompting은 LLM 능력의 **하한선**만 측정. CoT가 수행 가능 태스크 집합을 확장 → H1의 "분해 = 능력 확장" 주장 실증.
- Ablation 4종(equation only / variable compute / chain after answer / standard) 모두 CoT보다 열등 → **자연어 순차 추론 단계 자체**가 효과의 핵심. 중간 단계의 질적 기여 실증.

⚠️ **교재 밖 자료**:
- "실제 추론"인지 통계적 패턴 매칭인지 미검증 — Wei et al. 본인 인정. [[declarative_procedural_LLM]] · [[AI_잠재공간_기억철학]] 연결.
- ≥100B 창발 = 현재 Claude Sonnet/Opus급 외 소형 모델에 H1 매핑 적용 불가.
- 잘못된 체인에서도 우연 정답 가능 (50개 중 2개) → 분해 + 검증 이중 구조 필요.

**Zhou et al. (2023) 와의 관계**: Least-to-Most prompting은 CoT의 직계 확장 — 하위 문제 분해 → 순차 해결. Wei et al. CoT = H1 행 1 원전; Zhou et al. LtM = H1 행 2(촉구 위계 fading) 직접 차용.

### 추론 단계 매핑 확장 후보 (박상언 2023, ToT 경유) — 교재 밖

[[P2023_박상언_LLM추론_프롬프트엔지니어링|박상언·강주영 (2023)]] 의 LLM 추론 서베이는 *Tree-of-Thought (ToT)* 의 핵심 구조를 **사고 분해 → 사고 생성 → 사고 평가** 3단계로 정리. ToT는 CoT의 *선형 분해*에 *분기 + 평가* 단계를 추가 — H1 분해 차원에 *내부 추론 평가* 추가:

| ABA 절차 | [[CTA_인지적과제분석\|CTA (CPP)]] | ToT (박상언 2023) | 기존 매핑 |
|---|---|---|---|
| 행동 단계 | 활동 단계 (AC) | 사고 분해 | CoT step 분해 |
| (해당 없음) | 의사결정 단계 (DC) | 사고 평가 (분기 결정) | 분기 조건 |
| (해당 없음) | 개념적 지식 (CO) | 사고 생성 (도메인 적용) | 도메인 컨텍스트 |
| (해당 없음) | 품질 기준 (QS) | 사고 평가 (전반) | 검증 기준 |

→ ToT의 *사고 평가* 단계가 CTA의 *의사결정 + 품질 기준* 두 항목과 맞물림. CoT는 ABA 행동 분해와 가깝고, ToT는 CTA처럼 *분해의 풍부한 정밀화*. 한국어 학술 풀에 *분해 깊이의 단계적 차이* 명시.

⚠️ **교재 밖 자료 + 반증 후보**:
- ToT는 *모델 내부 추론*에 한정. ABA·CTA는 *학습자·전문가 행동·인지*. 작용 차원이 다름 (모델 내부 vs 외부 행동/인지).
- 박상언 본인이 §11에서 ReAct·Plan-and-Solve 미포함을 한계로 인정. 매핑이 *2023년 12월까지의 기법 한정* — 후속 기법 보충 필요.
- 박상언 §9 비판: arXiv 의존, 정량 비교 부재. 인용 시 유보 표현 의무.

### 포괄적 서베이 근거 허브 (Schulhoff et al. 2024) — 교재 밖 ✅ 볼트 정식 등재 완료 (2026-05-04)

[[P2024_Schulhoff_프롬프트엔지니어링_서베이|Schulhoff et al. (2024)]] *The Prompt Report: A Systematic Survey of Prompting Techniques* (ICLR 2024 / arXiv:2406.06608)는 PRISMA 기반 체계적 문헌 검토로 **58개 텍스트 기반 프롬프팅 기법**을 분류한 80페이지 포괄 서베이. H1 핵심 근거 논문들(CoT·LtM·ToT)이 모두 이 서베이의 **Decomposition 범주(§2.2.3)**에 함께 집결되어, H1의 "분해-기반 기법군" 주장을 학계 표준 분류 체계로 뒷받침.

| Schulhoff §2.2.3 Decomposition | H1 ABA 대응 |
|---|---|
| Least-to-Most (Zhou et al. 2022) | 과제분석 2단계 + 촉구 위계 fading |
| Tree-of-Thought (Yao et al. 2023) | 문제공간 탐색 (BFS/DFS) |
| DECOMP (Khot et al. 2022) | 기능별 하위과제 분배 |
| Plan-and-Solve (Wang et al. 2023) | 계획-실행 2단계 |
| Metacognitive Prompting | 5단계 메타인지 과정 |

**추가 H1 근거**:
- §1.2.2 Prompt Chain 정의: "이전 출력이 다음 템플릿을 파라미터화" = **행동연쇄(behavior chaining)** 공식 정의와 구조 동형
- Figure 2.11 인용 빈도 차트: Least-to-Most(6위), ToT(11위) — 분해 기법의 학계 영향력 가시화
- §2.2.1 ICL / Few-Shot 설계 6요인 = **촉구 위계 설계** 원칙 (양·순서·유사성·레이블 품질·형식) 과 대응

⚠️ **교재 밖 자료**: R002/교재밖 라벨. 관찰적(observational) 입장 — 기법 유효성 주장 없음.

### ICL 정의·분류 체계 허브 (Dong et al. 2024) — 교재 밖 ✅ 볼트 정식 등재 완료 (2026-05-04)

[[P2024_Dong_ICL_서베이|Dong et al. (2024)]] *A Survey on In-context Learning* (EMNLP 2024, pp. 1107–1128)는 이 서베이 이전 H1에 등재된 **Wei (2022) CoT · Zhou (2023) LtM** 을 포괄하는 상위 분류 체계(Taxonomy)를 제공. ICL을 "파라미터 업데이트 없이 시연(demonstration)만으로 과제 학습"으로 공식 정의하고, 훈련(pretraining/warmup) × 추론(demonstration/instruction/scoring) × 분석의 2단계 taxonomy로 체계화. → [[in_context_learning]] 개념 노트 신규 작성.

| ICL 구성요소 | ABA / CTA 인접 구조 | 공통 구조 |
|---|---|---|
| ICL 공식 정의: "시연(demonstration)으로 학습" | ABA 시범(modeling) / 모방학습 | 선행 사례를 보고 패턴을 내재화 |
| Demonstration Selection (§4.1.1) — 유사 예시 우선 | 촉구 위계 설계 — 현재 기술 수준에 맞는 예시 제공 | 학습자(모델) 현재 수준 기반 시범 선택 |
| Demonstration Ordering: 단순→복잡 (ICCL) | 과제분석 — 선행 기술 → 목표 기술 순서 + 촉구 fading | 완전촉구 먼저, 독립 수행으로 점진적 이행 |
| Instruction Formatting: CoT + LtM (§4.2) | ABA 과제분석 + 촉구 위계 fading | (이미 H1 매핑 표 행 1·2에서 확인; ICL taxonomy가 재확인) |
| Scoring Function (§4.3): 모델 반응 평가 방법 | ABA 강화 스케줄: 반응 후 결과 제공 방식 | 반응 품질에 따른 차등 처우 |
| Warmup 훈련(§3.2): inference 전 ICL 특화 tuning | ABA 사전 기술 훈련 / 매칭 샘플 훈련 | 목표 맥락 진입 전 준비 훈련 |

**H1에 주는 추가 근거**:
1. **ICL 분류 허브**: H1에 이미 등재된 CoT(Wei 2022)·LtM(Zhou 2023)이 Dong et al. taxonomy §4.2 Instruction Formatting 하위로 집결 → H1 "분해-기반 기법군" 주장이 ICL 필드 공식 taxonomy에서도 확인됨 (Schulhoff 2024 Decomposition 범주에 이어 **두 번째 taxonomy 확인**).
2. **"유추를 통한 학습"**: "The key idea of in-context learning is to learn from analogy" (Dong et al. p.1107) — ABA의 시범(modeling)과 모방(imitation) 기반 학습이 "유추(analogy)"라는 동일 논리로 기술됨.
3. **Demonstration selection이 모델 의존적**: 최적 시범 선택이 학습자(모델) 특성에 따라 달라짐 (Table 2) — ABA의 "학습자 수준 개별화" 원칙과 동형.
4. **Long-context ICL 한계**: 시연 수 증가가 성능 보장 않음, 오히려 저하 가능 — ABA의 "촉구 과잉(prompt dependency)" 문제와 기능적 유사성.

⚠️ **교재 밖 자료**: R002/교재밖 라벨. ICL 개념은 Cooper 교재·이성봉(2019)에 없음. 강 교수 자문 미수렴. **단정 금지**.

### ABA 원전 교재 직접 인용 — 과제분석·행동연쇄·촉진위계 1차 학술 지원 (정선화·이소현 2006) — ABA 정통 ✅ 볼트 정식 등재 완료 (2026-05-04)

[[P2006_정선화_자폐_통합교수전략|정선화·이소현 (2006)]] *통합 상황에서의 자폐 범주성 장애 학생을 위한 교수전략 고찰* (특수교육 제5권 제1호, pp.91-111)은 통합 환경 ABA 교수전략 5대 범주를 체계적으로 정리한 한국어 문헌 리뷰. **Cooper·Heron·Heward (1987) *Applied Behavior Analysis* 원전 교재를 직접 인용** — H1의 "ABA 과제분석 = 복잡한 기술을 작은 단계로 나누어 가르치는 절차" 주장의 **1차 ABA 정통 학술 지원**.

| 정선화·이소현 (2006) 기술 | H1 매핑 차원 | ABA 원전 (Cooper et al. 1987) |
|---|---|---|
| 과제분석 (task analysis) §4.3 — "복잡한 기술을 학습할 수 있도록 작은 단계로 나누어서 교수의 내용과 연속성을 결정하는 접근" | 과제분석 ↔ 작업 분해 / CoT 분해 (H1 행 1) | Cooper·Heron·Heward (1987) 직접 인용 |
| 행동연쇄 — "각 단계가 다음 단계의 자연적 단서(natural cue)로 기능" | 행동연쇄 ↔ 에이전트 워크플로우 (H1 행 3): 앞 출력이 뒤 입력의 변별자극 | 연쇄(chaining) 정통 정의 |
| 촉진 위계 §4.1 — 언어적·시각적·시범·신체적 도움 → 점진 소거 → 독립 수행 | 촉구 위계 ↔ few-shot → zero-shot fading (H1 행 2) | 촉진(prompting) + fading 정통 절차 |

**H1에 주는 함의**:
- 지금까지 H1의 "과제분석 = 작업 분해" 대응은 **Zhou et al. (2023) 의 least-to-most 용어 차용**을 통해 간접 확인되었으나, ABA 교재 정통 정의가 볼트에 없었음. 본 사료는 Cooper et al. (1987) 과제분석 정의를 한국어로 체계화한 **1차 한국 학술 지원**.
- "촉진 각 단계가 다음 단계의 자연적 단서로 기능" 구절은 H1의 "앞 행동(출력)이 뒤 행동(입력)의 변별자극이 됨"과 ABA 어휘 수준에서 **직접 일치** — 행동연쇄 ↔ 에이전트 워크플로우 매핑의 ABA 정통 근거.

## 반론·반증 가능성

### 기각 조건
1. **구조적 대응이 깊지 않다는 반박**: ABA의 강화 메커니즘은 학습자(행동 주체)의 동기·내적 상태에 작용하는데, LLM은 매 호출마다 무상태(stateless)이므로 "강화 스케줄 = RLHF"는 표면적 비유일 뿐이라는 비판이 가능. 이 비판이 학술적으로 정합되면 H1의 강도는 약화.
2. **방법론 차이**: ABA의 핵심은 단일대상연구(single-subject design) 방법론에 있는데, 이는 프롬프트 엔지니어링에 직접 대응되지 않음. "절차적 유사성 ≠ 방법론적 유사성".
3. **ABA 7차원 (Baer, Wolf, Risley 1968) 적용 시 일관성 부족**: 7차원 중 "응용적(applied)·행동적(behavioral)·분석적(analytic)" 같은 학문 정체성 차원이 프롬프트 엔지니어링에는 없거나 다른 형태로 존재. 매핑이 부분적이면 H1은 약한 형태로만 유지 가능.

### 이미 발견된 반증 후보
- (아직 문헌 미정독, 추가 탐색 필요)
- **CTA 매개 매핑의 전이 손실 위험** — ABA→CTA→프롬프트로 매개하는 경로는 ABA의 **단일대상연구·강화 메커니즘** 같은 핵심 정체성을 우회하므로, 매핑이 풍부해지는 대신 ABA 측 정합성이 약화될 수 있음. CTA로 확장하면 H1이 더 이상 *ABA에 대한* 가설이 아니게 될 가능성. ([[CTA_인지적과제분석]])

# H2 — 프롬프트 엔지니어링 담론에 아직 수입되지 않은 ABA 원리가 존재한다

## 주장

ABA가 가진 원리 중 **자극통제(stimulus control)**·**동기조작(motivating operations)**·**일반화 절차(generalization procedures)**는 현재 프롬프트 엔지니어링 담론에서 명시적으로 다뤄지지 않고 있으며, 이를 도입하면 프롬프트 설계 실천에 새로운 시각을 제공할 수 있다.

> **본 가설은 H1(구조적 대응)이 부분적으로만 성립한다는 전제 위에 선다.** 즉 H1의 매핑이 완전하다면 H2는 성립하지 않는다. H1의 "대응이 깨지는 지점"이 H2의 후보가 된다.

## 배경

[[강수정_2026-04-24_답신]]에서 강 교수가 "특수교육에서 공학적 접근 시도가 그간 많이 이루어지지 않았기에 필요한 연구"라 평가한 것의 역방향 함의: **공학(프롬프트 엔지니어링) 측에서도 특수교육의 자산을 충분히 흡수하지 못하고 있다**.

## 후보 원리 — 누락 가능성 검토

### 자극통제 (Stimulus Control)
- **ABA에서**: 특정 선행자극(SD) 하에서만 특정 행동이 강화되도록 설계 → 행동의 맥락 의존성을 명시적으로 통제.
- **프롬프트 엔지니어링에서**: "system prompt"·"role assignment"가 부분적으로 같은 기능. 그러나 자극통제의 정밀도(예: 변별자극 SΔ vs SD 구별, fading 절차)는 미수입.
- **만약 수입한다면**: 프롬프트의 컨텍스트 설정을 "자극통제 설계"로 재기술 → 어떤 단서가 어떤 출력을 끌어내는지 의도적·체계적으로 통제하는 메소드.
- **외부 사례 후보 ([[객체개수불일치]] — 교재 밖)**: [[P2026_허준_StableDiffusion_프롬프트엔지니어링|허준·연규필 (2026)]] 의 객체 수량 재현 실패는 *프롬프트 토큰 ‘three’가 변별 자극으로서 충분한 자극 통제를 발휘하지 못하는 사례*로 매핑 후보. 단, 모델 측 한계는 *훈련 데이터 분포 편향*이고 ABA 자극통제는 *유기체-환경 학습*이라 매핑은 *기능적 동형*에 그칠 가능성 (반증 후보).
- **자극 통제의 비율적 정밀화 사례 ([[ROSES_프롬프트_프레임워크|ROSES Visual Hierarchy]] — 교재 밖)**: [[P2026_이명원_ROSES_포스터디자인_프롬프트|이명원·이현수 (2026)]] 의 *Refined ROSES* 는 60-30-10 규칙·Z/F 패턴·게슈탈트 원리 등을 프롬프트에 명시화하여 DALL-E 3의 시각 산출을 *비율적·구조적으로 통제*. **시각적 단서가 명확할 때 시선 반응이 어떻게 유도되는가**라는 자극 통제 어휘로 재기술 가능. 본 사료에서 레이아웃 균형 +12.4% / 서사 일관성 +10.6% 향상은 *자극 통제 정밀화의 효과*에 대한 정량 증거. 단, Case 3·5 -0.3% 결과 → 자극 통제 효과의 *콘텐츠 의존성* (반증 후보).
- **수치적 명확성 사례 ([[P2023_이명숙_ChatGPT_프롬프트엔지니어링|이명숙 2023]] 원칙 2 — 교재 밖)**: '짧게 표현해줘' → '2줄로 표현해줘' 같은 *수치적 명시화*가 ChatGPT 결과의 정합성을 높인다는 비전공자 수업 관찰. 60-30-10 규칙(이명원·이현수)과 동일 결의 *변별 자극(SD) 정밀화의 수치 명시*. 단, 학회 발표논문 4쪽으로 통계적 검증 없음 — 가설 단계 (반증 후보).
- **메타 차원 사례 — 연구자 능력의 외재화 ([[P2026_한승주_LLM_질적연구|한승주 (2026)]] — 교재 밖)**: 한승주의 *이론적 민감성 학습*(Glaser & Strauss, 1967; Birks & Mills, 2015) 개념은 *연구자 자신의 분석 능력*을 시스템 프롬프트로 *체계화·인코딩*하는 시도("질적 연구자의 보조 코더로서 LLM 활용은 연구자 자신의 이론적 민감성을 프롬프트로 정교하게 체계화하는 것" p.227). ABA 어휘로는 *변별 자극(SD) 인코딩 + 자극 통제 시스템 프롬프트화 + 강화 이력 누적*으로 재기술 가능. ⚠️ **차원 차이 강함** — 위 허준·ROSES 사례가 *모델 출력 측 자극 통제* 라면, 본 사례는 *연구자 분석 능력의 시스템 프롬프트 외재화* — H2의 메타 사례. [[LLM_활용_두흐름|코더로서 LLM]] 흐름의 자기 인식이기도 함. ⚠️ **반증 후보**: 질적 연구자의 *능력*과 ABA의 *변별 자극*은 작용 차원(연구자 인지 vs 학습자 행동)이 달라 *기능적 동형*에 그칠 가능성. 강 교수 자문 가치 영역.
- **환각 5원인 ↔ 자극 통제 / 자극 일반화 실패 ([[P2024_안진호_생성형AI_환각최소화|안진호·정욱환 (2024)]] — 교재 밖)**: 본 사료의 [[환각_5원인]] 중 (3) 질문 모호성·(5) 사용자 입력 영향은 ABA 어휘로 **변별 자극(SD) 불충분 / 유도된 변별 학습**으로, (1) 훈련 데이터 한계·(4) 예측 기반 생성은 **자극 일반화 실패** ([[자극일반화]])의 LLM 변종으로 재기술 가능. R002 매핑 작업 시 LLM 보조 도구 운영 체크리스트로 활용. ⚠️ **반증 후보 + 사료 결함 명시**: ABA 자극일반화는 *유기체-환경 학습*이고 LLM 환각은 *모델 내부 확률 분포 한계* — 작용 차원 차이 강함. 또한 본 사료 자체에 학술적 결함 4건(H4 진술 오기·공선성·R² 표기 오류·이해관계 미고지·선택 편향) — 인용 시 유보 표현 의무.
- **작화(confabulation) — 환각 의인화 비판 + 정보 공백 메우기 ([[P2025_김대길_AI작화증_충성도|김대길 (2025)]] — 교재 밖, contradicts [[P2024_안진호_생성형AI_환각최소화]])**: 김대길은 *환각*을 *의인화 오류*로 비판하고 [[작화_vs_환각|작화 (Confabulation)]] 로 재정의. ABA 어휘로는 *변별 자극 부족 시 모델이 자체 보완 서사 생성* — 자극 통제 실패의 한 양상. 인접 [[인지적_정서적_신뢰_이원화|이중 신뢰]] 구조에서 *정서적 신뢰가 인지적 검증을 우회*하는 [[인지적_구두쇠_역설충성도|역설충성도]] 메커니즘은 H2 *동기조작 미수입*의 LLM 변종 사례. ⚠️ 본 사료 결함 4건 — 인용 유보 의무.
- **Ayres 감각통합 (SI) = ABA 자매 이론 ([[P2026_전익환_자폐_공간환경|전익환·이선정 (2026)]] — 교재 밖, ⭐ 가장 강한 후보)**: [[감각통합_이론_SI|Ayres SI]] (1972, 1979)는 ABA와 동시대 자폐 중재 이론 — 작업치료 학파. *환경 변수 통제* 관점은 ABA *변별 자극(SD)*과 동일하나 *명명·이론 출처*가 다름. 4 사례 분석에서 [[5차원_공간평가_프레임워크|5차원 평가 프레임]] 도출. [[전이_건축학|전이 건축학]]·[[감각_스캐폴딩|감각 스캐폴딩]]·[[회피_공간|회피 공간]] 모두 ABA *자극통제·일반화 절차*의 공간 차원 사례. ⚠️ 강 교수 자문 가치 매우 강함 — *기능적 동형* vs *학파 분리*.
- **능력 ↔ 실수행 괴리 — 동기조작 임상 사례 ([[P2026_Chiu_IADL_조현병|Chiu & Lee (2026)]] — 교재 밖)**: [[능력_실수행_괴리|capacity-performance gap]] 가설의 임상 정신의학 측 동의 자료. *avolition (의욕저하)* = ABA *동기조작 미설정*의 임상 표현. 측정 도구 정밀화([[Rasch_단일차원성_검증]]·[[Cronbach_α_사용분기]]) 작법은 R002 매핑 검증의 메타 자원으로 차용 가능.
- **변별 자극 어휘의 *유연 적용* — Flexible Idiom 패턴 ([[P2026_지현정_노이로제_정신의학사|지현정 (2026)]] — 교재 밖)**: ABA 정통은 변별 자극(SD)에 *단일 의미·일관 적용*을 전제하나, LLM 프롬프트 현장에서는 어휘가 [[Flexible_Idiom_진단어휘|flexible idiom]] 으로 작동 — 도메인·맥락·사용자에 따라 다중 의미·다중 적용. *유연 변별 자극* 가설 (working): "프롬프트 어휘는 단일 SD가 아니라 flexible idiom으로 작동할 때 비전공자 접근성이 높아지지만, 정밀성은 약화된다." 본 매핑은 H2 자극통제 영역에 *언어 차원* 자원 도입. ⚠️ 차원 차이 강함 — 노이로제는 *임상 진단어휘*, ABA SD는 *행동 통제 자극*. 강 교수 자문 가치 영역.
- **권위의 번역적 구축 — 메타 차원 자가-감시 ([[P2026_지현정_노이로제_정신의학사|지현정 (2026)]] — 교재 밖)**: ABA→프롬프트 *번역* 시 누가 권위를 가지는가 라는 질문이 H2 매핑 작업 자체에 적용 ([[전문가권위의_번역적_구축]]). 본인이 단계 3 *번역자* 위치 → 단계 4 *권위 형성자*로 전환되는 점에서 R002 제5원칙(디자인교육 전공 관점) 의 자가-감시 절차 신설 자원. 강 교수 자문 가치 매우 강함.
- **인구사회학 단독 조건의 변별 자극 결핍 ([[P2026_양소은_LLM도플갱어_자아구성|양소은 외 (2026)]] — 교재 밖)**: 50명 LLM 도플갱어 실험에서 *조건 1 (인구사회학 단독)* 은 *고정관념적 응답* 으로 수렴. ABA 어휘로 **변별 자극 결핍 → 자극 통제 실패 → 고정관념 일반화**. [[정체성_3차원_위계]] 의 단계적 추가가 SD 풍부화로 작용. ⚠️ 단 [[숨긴_자아_LLM_추론한계|숨긴 자아]] 영역에서는 SD 풍부화 효과 한계 — *변별 자극의 본질적 접근 한계* 명시.
- **변별 자극의 메타 분류 — 조하리의 창 4 영역 ([[P2026_양소은_LLM도플갱어_자아구성|양소은 외 (2026)]] — 교재 밖)**: [[조하리의_창_4영역]] 4 영역 (드러낸·숨긴·맹점·미지) 은 SD 의 **접근 가능성 메타 분류**. ABA 자극통제는 *외부 관찰 가능 SD* 전제 → *드러낸 자아·맹점* 영역만 작동. *숨긴 자아·미지* 영역은 SD 부재로 자극통제 원리 자체로 접근 불가. H2 의 *변별 자극 풍부화* 전략에 *영역별 작동 한계* 경계 조건 추가.
- **ABA 동기조작 (MO) 의 LLM 변종 — 정체성 가중치 조정 ([[P2026_양소은_LLM도플갱어_자아구성|양소은 외 (2026)]] — 교재 밖)**: [[정체성_가중치_조정]] (조건 4) 은 ABA *동기조작* 의 LLM 부분 수입 사례. 한 차원 가중 ↑ → 그 차원 응답 빈도 ↑ = MO *박탈/포만* 의 비례 조작 변종. ⚠️ 차원 차이: ABA MO 는 *유기체-환경 학습 변형*, LLM 가중치 조정은 *모델 입력 비례 조작*. 양소은 본문은 ABA 인용 없음 → *기능적 동형* 강함, *학파 분리* 의무.

### 동기조작 (Motivating Operations, MO)
- **ABA에서**: 강화제의 가치를 일시적으로 증감시키는 환경 조작 (예: 박탈 → 강화제 가치 상승).
- **프롬프트 엔지니어링에서**: 거의 미언급. 가장 가까운 것이 "model temperature"와 "guidance" 정도지만 동기조작 개념은 부재.
- **만약 수입한다면**: LLM 호출 시점의 "가치 함수 조작"이라는 새로운 차원 — 예컨대 "이 응답이 사용자에게 왜 중요한지" 명시가 출력 품질에 영향을 주는지 실증 가능.

### 일반화 절차 (Generalization Procedures)
- **ABA에서**: 학습된 행동이 다른 자극·환경·시간으로 전이되도록 의도적으로 훈련 (예: multiple exemplar training, indiscriminable contingencies).
- **프롬프트 엔지니어링에서**: "domain transfer"·"out-of-distribution"의 문제는 인식되어 있으나, ABA 수준의 의도적·절차적 일반화 훈련법은 미수입.
- **만약 수입한다면**: few-shot 예시 설계에서 "다양한 사례를 의도적으로 분산 배치하는 절차"를 명시화 → 일반화 가능성 향상.

### 자기관리·shaping (Self-Management / Shaping) — Self-Refine 경유 외부 사례
- **ABA에서**: 자기관리(self-management)는 학습자가 자기 행동을 스스로 모니터·평가·강화. shaping은 외부 코치가 학습자의 점진적 근사 반응을 강화.
- **프롬프트 엔지니어링에서**: [[Self-Refine_프롬프트]] (Madaan et al., 2023) 패턴 — LLM이 자신의 초안을 진단·재작성하는 루프. 명시적으로 ABA 자기관리·shaping을 인용하지 않음.
- **외부 사례 후보 ([[P2026_허준_StableDiffusion_프롬프트엔지니어링|허준·연규필 2026]] — 교재 밖)**: Self-Refine을 Stable Diffusion 객체 수량 재현에 적용해 정확도 0% → 27.06% 도달. *기능적으로* 자기관리·shaping과 유사한 평가-피드백-재시도 루프.
- **만약 수입한다면**: Self-Refine 루프 설계를 ABA 자기관리·shaping 어휘로 재기술 → 평가 기준·강화 단위·점진적 근사 단계의 명시화 가능.

⚠️ **반증 후보 (위 자기관리·shaping 매핑)**: LLM은 매 호출마다 stateless이므로 ABA의 *시간적 학습 누적*이 결여. Self-Refine이 "행동의 평가자와 행동 주체가 같다"는 자기관리의 핵심 속성을 정말 만족하는지(피드백 LLM이 진정으로 자기 출력을 평가하는지 vs 별개의 함수 호출인지) 검토 필요.

### 시간지연 (Time Delay) — ABA 고유 자극 통제 전이 절차, 프롬프트 엔지니어링 미수입 (정선화·이소현 2006 §4.3) — ABA 정통 ✅ 볼트 정식 등재 완료 (2026-05-04)

[[P2006_정선화_자폐_통합교수전략|정선화·이소현 (2006)]] §4.3 *시간지연(time delay)* 은 ABA의 고유 **자극통제 전이(transfer of stimulus control)** 절차:

| 단계 | 절차 | ABA 어휘 |
|---|---|---|
| 0초 지연 | 목표 자극 → 즉시 촉진 제공 (교사 주도 반응 유도) | SD + 즉시 촉진 → 정반응 → 강화 |
| 점진적 지연 증가 | 목표 자극 → 일정 시간 대기(1초→2초→…) | SD → [지연 간격] → 학생 자발 반응 대기 |
| 독립 수행 | 자발 반응이 지연 내 발생 시 촉진 불필요 | SD만으로 반응 유발 = 자극통제 전이 완성 |

**H2와의 연결**: 시간지연은 프롬프트 엔지니어링 담론에서 명시적으로 다뤄지지 않는 ABA 고유 절차. "제로-촉진 → 점진 지연 → 독립 수행"의 타이밍 기반 설계는 few-shot fading(H1)의 *타이밍 차원 확장*이지만, 프롬프트 엔지니어링에서 **지연 시간 자체를 변수로 설계하는 절차**는 부재. H2 자극통제 "프롬프트 엔지니어링에 아직 수입되지 않은 원리" 후보.

⚠️ **반증 후보**: LLM 호출은 동기 I/O — "지연"의 맥락이 ABA 물리적 시간 지연과 다름. "컨텍스트 제공 후 대기" 개념으로 재기술해야 의미 있는 전이 가능. **ABA 정통 교재 Cooper et al. (1987) 2차 인용** — 원전 직접 정독 필요.

### 고확률요구 순서 (High-Probability Request Sequence) — 동기조작 선행자극 조작의 한국어 정통 사례 (정선화·이소현 2006 §4.5) — ABA 정통 ✅ 볼트 정식 등재 완료 (2026-05-04)

[[P2006_정선화_자폐_통합교수전략|정선화·이소현 (2006)]] §4.5 *분산 요구(high-probability request sequence)* — 어려운 과제 전 쉬운 과제(준수 이력이 높은 요구) 를 3~5개 선행 제시하여 **반응 모멘텀(behavioral momentum)** 형성 후 목표 과제 수행 유도.

**H2 동기조작 섹션과의 연결**:
- H2 §동기조작에서 MO는 "강화제의 가치를 일시적으로 증감"으로만 정의됨. 고확률요구 순서는 MO의 *선행자극 조작 변형* — 성공 이력으로 반응 동기를 사전 상승.
- 프롬프트 엔지니어링에서 유사 직관: "쉬운 단계부터 LLM에게 물어보고 점점 어려운 질문으로 유도" — 그러나 ABA의 반응 모멘텀 이론에 명시적으로 기반한 설계는 부재.
- [[P2023_Zhou_LeastToMost_프롬프팅|Zhou et al. (2023) LtM Prompting]] 은 구조적으로 유사(쉬운 하위 문제 → 어려운 문제)하지만 **반응 모멘텀·MO 어휘는 인용하지 않음** → H2 기각 조건 3("이름은 다르나 같은 개념") 부분 해당 가능성. 다만 LtM은 분해 + 순차 해결이고, 고확률요구 순서는 *성공 경험 누적 → 동기 상승*이라는 **기능적 목적이 다름**.

⚠️ **반증 후보**: 반응 모멘텀 효과의 LLM 적용 가능성은 미검증. LLM은 상태 비보존(stateless) → 이전 "성공 경험"이 다음 호출에 내재화되지 않음. 프롬프트 내 선행 쉬운 예시 제공이 동기조작 효과를 발휘하는지 실증 필요.

### Pak 외 챗봇 톤 = 자극 통제의 정량 사례 (박윤수 외 2026 §3 인용) — 교재 밖

[[P2026_박윤수_AI환각_교육적대응|박윤수·박호현·이유미 (2026)]] 가 인용한 Pak 외 (2024) 의 196명 (공립대 71+군사학교 125) 챗봇 톤 실험은 자극 통제 매핑의 **가장 강한 정량 증거**:

- 변별 자극(SD): 챗봇 응답 톤 (중립 vs 정중 polite)
- 측정 행동: 잘못된 정보(환각 포함) 수용 여부 (다지선다형 정답률)
- 결과: **정중 톤 → 환각 수용 가능성 ↓ (보수적 편향 conservative bias)**

→ ABA 어휘로 재기술: 정중 톤 챗봇 = *경계·검증 행동을 유도하는 SD*. 학습자의 *환각 식별 반응*이 SD 변화에 따라 달라짐 = 명백한 자극 통제 효과. 이는 H2의 *역방향 사례* — **챗봇 톤 = 시스템 프롬프트 ≈ 자극 통제 설계**가 이미 일부 적용되고 있음을 시사.

⚠️ **반증 후보**:
- 본 사료는 Pak 외 *2차 인용*. 원논문 직접 정독 필요.
- 정중 톤이 보수적 편향을 유도하는 것이 *자극 통제*인지 *사회적 단서*인지 분리 어려움.

### RAG = 외부 자극 통제 보조 (박윤수 외 2026 §5 결론) — 교재 밖

[[RAG_검색증강생성|RAG]] (Retrieval-Augmented Generation)는 LLM이 응답 생성 전 외부 데이터베이스를 검색하여 그 결과를 컨텍스트로 활용하는 패턴. **외부 단서로 변별 자극을 보강**하는 자극 통제 보조 메커니즘:

| RAG 단계 | ABA 매핑 (working) |
|---|---|
| 사용자 프롬프트 | 1차 SD |
| 외부 검색 | SD 보강 단서 / 시각 보조 자료 / 촉구 (visual prompt) |
| LLM 생성 | 행동 |
| 응답 정확도 ↑ | 자극 통제 정밀화로 반응 정확도 ↑ |

박윤수 외 (2026) §5 제언: "RAG 기반의 생성형 인공지능의 교육적 활용에 대해서 보다 많은 연구가 수행해야". → H2 기각 조건 3 ("이름은 다르나 같은 개념") 의 부분 사례 — **RAG는 자극 통제 어휘로 재기술 가능하지만 본 사료가 ABA를 인용하지 않음**.

⚠️ **반증 후보**: ABA 촉구(prompt)는 *fading* 메커니즘이 핵심 (학습자가 점진적으로 의존하지 않도록). RAG는 *항상 외부 검색을 사용* — fading 결여. 자극 통제 기술적 차이.

### 행동 측정자(observer) — VLM-as-Judge 경유 외부 사례
- **ABA에서**: 행동 측정자는 학습자 행동을 부호화·평가. IOA(interobserver agreement) ≥ 80% 같은 재현성·객관성 기준이 *방법론의 핵심*.
- **프롬프트 엔지니어링에서**: [[VLM-as-Judge]] 패턴(시각-언어 모델이 AI 산출 평가) 이 외부 평가자로 사용되나, ABA observer 의 IOA·재현성 검증 절차는 미수입.
- **외부 사례 후보 ([[P2026_이명원_ROSES_포스터디자인_프롬프트|이명원·이현수 2026]] — 교재 밖)**: GPT-4o 를 평가자로 4 지표 5점 리커트 척도로 산출 평가. 본 사료 §5.2: VLM 평가자가 인간 전문가와 *간극* 있음 인정 — ABA IOA 기준 미충족 가능성.
- **만약 수입한다면**: VLM-as-Judge 의 IOA 측정·다중 평가자 합의·재현성 보고를 ABA 측정 방법론으로 재기술 → AI 평가의 객관성·신뢰도 기준 정립.

# H4 — 프롬프트 분해 효과의 임계점·콘텐츠 의존성·동질화 부작용

> **상태**: active (2026-05-03 격상 — 옵션 B 채택: status active, confidence 0.4 유지). 강 교수 자문 후 confidence 격상 가능.
> **근거**: 외부 학술 자료 4편 누적 패턴 (권동현·허준·이명원·최지연) + 한승주의 *평균적 수렴* 비판.

## 주장

프롬프트 분해의 정밀도와 산출 정확도의 관계는 **단조 증가가 아니라**:

1. **콘텐츠·도메인 의존성** — 분해 효과가 과제 특성에 따라 *비균질*하게 나타남. 어떤 영역에서는 강하고 어떤 영역에서는 약하거나 음(-)이 됨.
2. **임계점 존재 가능성** — 일정 정밀도 이상으로 분해를 정교화하면 효과가 정체·역전될 수 있음.
3. **동질화 부작용 (design slop)** — 정밀한 분해가 산출의 *다양성·창의성*을 감소시킴.

→ [[H1_과제분석_프롬프트분해_구조대응|H1]] "분해 정밀도와 산출 정확도의 구조적 대응" 의 *조건부 유보*. H1을 부정하지는 않되, *조건과 한계*를 명시하는 보조 가설.

## 배경

R002 외부 학술 자료 3편(권동현·허준·이명원)을 통합하면서 발견한 **3편 누적 의존성 패턴** + 한승주 (2026)의 *평균적 수렴* 비판이 본 가설의 동기.

## 근거

## ABA 매핑 후보 (working — 교재 밖)

H4가 성립한다면 ABA 측에는 다음 인접 자료:

- **자극일반화의 한계** ([[자극일반화]]) — 자극 통제의 정밀화가 *지나치면* 다른 맥락으로의 일반화가 약화. 프롬프트 분해의 정밀화도 같은 트레이드오프 가능.
- **rule-governed behavior의 한계** — ABA에서 인지된 한계: 환경이 빠르게 변하면 규칙이 오작동. 프롬프트 분해 = 규칙기반 통제이고, 콘텐츠 변화 시 같은 한계 발생.

⚠️ **반증 후보**:
1. **샘플 부족** — 3편 누적은 각 분야 사례 한두 개씩. 의존성 패턴이 일반화되는지 더 많은 사례 필요.
2. **이명원 Case 3·5의 -0.3%는 측정 노이즈일 가능성** — VLM-as-Judge IOA 미평가 (저자 자신 인정). 임계점 주장의 정량 근거가 약함.
3. **'design slop'은 디자인 영역 특수 용어** — 다른 도메인(텍스트·코드 생성)에서도 *동질화* 부작용이 발생하는지 별도 검증 필요.
