# Fitly 시연 영상 제작 가이드

> **편돌이들 팀 시연 영상 담당자용**
> 작성일: 2026-05-16
> 목적: 서비스 처음 보시는 팀원이 이 문서 한 장만 읽고도 영상 시나리오를 정확히 짤 수 있도록.
> 분량: 작동성 영상 3분 + 사용성 영상 2분 (해커톤 데모 영상 권장 시간 5분 정합)

---

## 0. 30초 요약 — Fitly가 뭔가요?

**Fitly는 한국 초등교사 임용 1차 시험 학습 플래너 PWA입니다.**

- **타겟**: 교대 4학년 ~ 졸업생, 매년 11월 임용 1차 응시자 (연 4만+ 명)
- **펀치라인**: *합격은 시간이 아니라 적합도다.*
- **핵심 가치**: 24년치 공개 기출(1997~2024)을 AI로 분류·분석해서 "**내 약점이 어디인지**"를 한 줄로 알려주고, 24년치 데이터에서 약점 영역만 골라 반복 학습시킴
- **수익 모델**: Free + Pass 49,000원/시즌 (박문각 인강 80~150만원의 5% 이하)

> 발표용 한 문장 — "24년치 데이터는 공개돼 있는데, 내 약점이 어디인지는 안 보였습니다. Fitly가 그 격차를 해소합니다."

## 1. 누구를 위해 만들었나 — 페르소나 김지원

영상에 등장할 가상 사용자입니다. 멘트·시나리오에 일관되게 등장시켜 주세요.

| 항목 | 내용 |
|---|---|
| 이름 | 김지원 |
| 신분 | 교대 4학년 |
| 응시 | 2026년 11월 임용 1차 |
| 월 지출 | 박문각 인강 12만원/월 + 종로 모의 9만원 |
| 이동 | 학교 → 학원 → 자취방, **왕복 1.5시간** |
| 페인 ① | **진단 부재** — 24년치 데이터가 공개돼도 내 약점이 어디인지 모름 |
| 페인 ② | **채점 모호** — 서술형 답안을 누가 어떻게 채점해주는지 불투명 |
| 페인 ③ | **이동 시간** — 학원 왕복 1.5시간이 매일 학습 시간 잠식 |

이 세 페인 → Fitly의 세 가치로 정확히 1:1 매핑됩니다 (영상 처음 30초에 깔리면 좋습니다).

## 2. 전체 페이지 지도 (사이드바 메뉴 순서)

좌측 사이드바에 **6개 메뉴**가 있습니다. 정확한 순서:

1. **대시보드** — 진입 즉시 보이는 홈
2. **학습 계획** — 시험일 역산 일일 목표
3. **기출 분석** — 24년치 기출 분포 + 분석 + 토픽맵 + 로드맵
4. **팟캐스트** — Gemini multi-speaker TTS 강의 오디오
5. **학습 분석** — 본인 학습 기록 추이·취약점
6. **마이 페이지** — 프로필 + 3 트랙 라이브러리 + 배지

사이드바 **하단**에는 **설정** + **로그아웃** 두 개가 더 있습니다.

그리고 **학습 트랙 3개** (`/study/quiz`, `/study/keyword`, `/study/mistake`)는 사이드바에 직접 노출되지 않고, 대시보드·학습 계획에서 진입합니다. 시연의 핵심은 사실 이 세 트랙 안에 있습니다.

## 3. 페이지·기능 친절 설명 (영상 담당자가 화면을 보면서 멘트할 수 있도록)

### 3.1 로그인 / 회원가입 / 온보딩

- **로그인 페이지** (`/login`) — 이메일·비밀번호 또는 Google OAuth
- **회원가입** (`/signup`) — 이메일·비밀번호
- **온보딩**: 회원가입 후 처음 진입 시 대시보드에 *"Fitly 첫 방문을 환영합니다"* 배너가 떠 있고, 두 단계 안내:
  1. 시험일 등록 → `/settings`
  2. 기출 분석 시작 → `/exam-analysis`

> **영상 멘트**: "가입은 이메일 하나로 30초. 첫 진입 시 두 단계 안내가 보입니다."

### 3.2 대시보드 (홈, `/dashboard`)

진입 즉시 첫 화면. 발표 슬라이드 9번 "사용성"의 *"첫 5초 가치"* 가 여기서 작동합니다.

구성:
- 상단 헤더: **"안녕하세요, {이름} 선생님."** + 펀치라인 부제 + 우측 "오늘 학습 시작" CTA · 알림 · 다크모드 토글 · 프로필 아이콘
- **KPI 카드 4종**: 목표·학습 진척도·최근 7일·연속 학습 일수
- **학습성과 추이 차트** (좌, 큰 영역) — recharts 기반 라인 차트
- **오늘의 학습 플랜** (우, 좁은 영역) — 풀이/키워드/오답 3 트랙 카드
- **취약 유형(Weak Types)** — 영역×인지수준 기반 약점 분석
- **AI 추천** — 가장 약한 영역 1순위로 무엇부터 풀라는 안내

하단 디스클레이머: *"본 대시보드의 KPI·차트·플랜은 본인 계정의 실제 학습 기록만으로 산출됩니다. 지역 교육청별 합격 컷·평균은 비공개이므로 Fitly가 보유하지 않습니다."* → 발표 슬라이드 7번 "정직성" 정합. 영상에서도 잠깐 비춰 주세요.

> **영상 멘트**: "진입 5초 안에 '내 약점이 어디인지' 한 줄이 떠야 한다 — 이게 정직성의 시작입니다."

### 3.3 학습 계획 (`/study-plan`)

**시험일 역산으로 오늘 해야 할 양을 자동 계산**합니다.

구성:
- 상단: "오늘의 복습 대기" (풀이 N장 · 키워드 N장 · 오답 N장) — 대시보드·마이페이지와 정합한 *좁은 정의* (이미 학습한 카드 중 due만 카운트)
- 일일 목표: 풀이·키워드·오답 트랙별 권장량
- 3 트랙 카드 (풀이 / 키워드 / 오답) — 각 트랙으로 진입 가능

> **영상 멘트**: "시험까지 D-180. 오늘 풀이 8장, 키워드 5장, 오답 3장. Fitly가 자동으로 계산합니다."

### 3.4 기출 분석 (`/exam-analysis`) — **발표 핵심**

**4 탭 구조**입니다. 사이드바 메뉴 클릭 → 페이지 진입 → 상단에서 탭 전환.

상단에는:
- KPI 4종 (2x2 grid): 시험지 수 · 문항 수 · 학년도 범위 · 시드 상태
- 내 지역 교육청 카드 (1차 합격선 추이, **검증된 데이터만** — verified=true)

탭:
1. **기출 시험지** (`?tab=papers`) — 1997~2024 시험지 카드 그리드. 신규/구버전 분리 표기.
2. **분석 (히트맵 3종)** (`?tab=analysis`):
   - **영역 × 학년도** — 11과목 + 교육학
   - **인지수준 × 학년도** — Bloom 6단계 (기억·이해·적용·분석·평가·창작)
   - **문항형식 × 학년도** — 객관식·단답형·서술형·논술형
3. **토픽맵** (`?tab=topic`) — 빈도 ≥ 2 키워드를 우선순위로 시각화 (네트워크 형태)
4. **로드맵 S/A/B/C** (`?tab=roadmap`) — 영역을 우선순위 등급으로 분류 (학습 순서 추천)

각 시험지 카드를 클릭하면 **시험지 상세 페이지** (`/exam-analysis/papers/[id]`) — 문항 본문 + 답안 + 검수 배지.

우측 상단에 **"인쇄 · PDF 저장"** 버튼 — 브라우저 인쇄 다이얼로그를 호출, PDF로 저장하면 파일명이 `Fitly 기출분석.pdf` / `Fitly 2024학년도 1교시.pdf` 식으로 떨어집니다 (2026-05-15 수정 완료).

> **영상 멘트**: "24년치 기출을 4개 탭으로 탐색. 영역별·인지수준·문항형식 분포를 한눈에 파악하고, 토픽맵에서 최빈 키워드를, 로드맵에서 학습 우선순위를 잡습니다."

### 3.5 풀이 트랙 (`/study/quiz`) — **시연 영상의 가장 큰 무대**

**Fitly의 진짜 차별점이 여기 있습니다.** 영상의 50% 이상을 여기에 써도 됩니다.

상단:
- "오늘의 복습 대기 — 풀이 N장 · 키워드 N장 · 오답 N장" 한 줄
- 트랙 스위치 nav (풀이 / 키워드 / 오답 + 각 due 카운트)
- 마크 필터 chip (북마크 / 별표 / 모르겠음)

**메인 영역 — SplitView (가장 중요!)**:
- **xl(≥1280px) 이상**: 좌·우 2분할
  - **좌(ProblemPane)**: 본문 + (채점 후) "내 답안" 카드 + 자가 채점 4 버튼
  - **우(AnalysisPanel)**: 채점 전 — 답안 입력 / 채점 후 — AI 분석 패널
- **xl 미만**: 세로 stack (모바일·태블릿)
- 좌우 비율 35% ~ 65% 사이로 드래그 가능

**풀이 흐름** (한 카드 안에서 모든 게 끝남):
1. 본문 검토 (좌)
2. 답안 입력 (우, 작문 영역)
3. **AI 채점** 버튼 클릭 → Gemini Flash 호출 → 분석 결과 5탭 노출
4. **자가 채점 4 버튼** (다시 · 어려움 · 좋음 · 쉬움) → ts-fsrs 기반 다음 출현 일자 자동 계산
5. 다음 카드로 자동 이동

**AI 분석 패널 5탭** (채점 후 우측에 나타남):
1. **총평** — AI가 답안을 종합 평가
2. **키워드** — 답안에서 누락/포함된 핵심 키워드
3. **diff** — AI 모범답안과 내 답안 문장 단위 비교
4. **답안 이력** — 같은 카드 과거 시도 기록
5. **신고** — AI 답안 오류 신고 채널

**시그니처 인터랙션** (작은데 강함):
- **키보드 1·2·3·4** — 자가 채점 4 버튼 단축키 (1: 다시, 2: 어려움, 3: 좋음, 4: 쉬움)
- **J·K** — 큐 안에서 이전/다음 카드 점프
- **마크 토글** (북마크·별표·모르겠음) — 우측 상단 3 아이콘
- **카드 태그** — 좌측 상단, 사용자 커스텀 해시태그
- **블라인드 모드** — 모범답안 숨기고 본인 답안만 검토
- **형광펜 (HighlightLayer)** — 본문에 드래그로 색칠
- **AI 학습 도우미 챗봇 (AssistantFab)** — 우측 중앙 floating 작은 원형 ✨ 버튼. 클릭 시 380px 챗 박스 펼침. 본문과 동시 상호작용 가능. 빠른 프롬프트 5종 ("이 문제 해설 줘", "키워드 정리" 등).
- **세션 진행 표시** — 풀수록 evergreen 바 채워짐

> **영상 멘트**: "본문 검토 → 답안 입력 → AI 채점 → 자가 채점. 카드 한 장 안에서 전 과정이 끝납니다. 키보드 1·2·3·4 단축키로 30장 연속 학습 가능합니다."

### 3.6 키워드 트랙 (`/study/keyword`)

풀이가 *문항 단위* 학습이라면, 키워드 트랙은 *개념 정리 노트* 학습입니다.

- 24년치 기출에서 **빈도 ≥ 2회** 등장한 핵심 키워드 **205개**를 추출
- 한 카드 = 한 키워드 + 시드된 정리 노트
- 진입 시 자동 reveal — 답안 입력 없이 노트를 바로 보고 자가 채점
- 채점 4 버튼 동일 (다시 · 어려움 · 좋음 · 쉬움)
- 형광펜·블라인드·마크 토글 동일

> **영상 멘트**: "기출에서 자주 나오는 205개 개념을 정리 노트 카드로. 핵심만 골라 빠르게 회독."

### 3.7 오답 트랙 (`/study/mistake`)

풀이 트랙에서 **"다시" 또는 "어려움"** 으로 채점한 카드가 **자동 합류**합니다. (개념 단순화: 별도 오답 등록 불필요.)

- 풀이 트랙과 동일한 SplitView + 자가 채점 흐름
- 상단에 **"오답노트 인쇄"** 링크 — 클릭 시 `/study/mistake-print` 페이지로 이동, 브라우저 인쇄 → PDF 저장하면 **`Fitly 오답노트.pdf`** 로 떨어집니다 (2026-05-15 수정 완료)

> **영상 멘트**: "다시·어려움 평가한 카드는 자동으로 오답 트랙에 합류. PDF로 인쇄해서 종이로도 회독 가능합니다."

### 3.8 팟캐스트 (`/podcast`) — **두 번째 차별점**

**Gemini multi-speaker TTS**로 자동 생성된 학습 오디오. 이동 중·운동 중·자기 전 청취용.

페이지 구성:
- 상단 **"즉시 생성"** (InstantGenerate) — 주제 입력 → 2인 화자 대화체 스크립트 → Gemini TTS → MP3 생성
- **Shared 에피소드** — 운영자가 시드한 공용 에피소드 (모든 사용자 청취)
- **User 에피소드** — 본인이 생성한 에피소드만 본인에게 노출 (RLS 정책)

에피소드 카드 클릭 → 상세 페이지 (`/podcast/[id]`) → 재생 버튼 → 우리 PWA 안에서 재생 시작

> **영상 멘트**: "내 약점 영역을 주제로 입력하면, 2인 화자 대화체 강의 오디오가 자동 생성됩니다. 통학 1.5시간을 학습 시간으로 전환."

### 3.9 학습 분석 (`/study-analysis`)

본인 학습 기록 기반 차트. 대시보드 KPI 의 *상세 버전*.

- 학습 추이 라인 차트 (recharts)
- 활동 히트맵 (1년 단위, GitHub contribution 스타일)
- 영역별·인지수준별 정답률
- 시간대별·요일별 학습 패턴

> **영상 멘트**: "내가 언제·얼마나·무엇을 학습했는지. 본인 데이터만으로 그려지는 차트."

### 3.10 마이 페이지 (`/me`)

내 프로필 + 학습 라이브러리 종합.

- 프로필 카드 (이름·이메일·가입일·목표 지역·시험일·D-day)
- 배지 (연속 학습·총 학습량 등 게이미피케이션 요소)
- **3 트랙 라이브러리 카드** — 풀이/키워드/오답 각각 보유 카드 수 + 복습 대기 수 (이건 대시보드·학습 계획과 동일한 수치)
- 최근 활동 (학습 세션 최근 5건)

### 3.11 설정 (`/settings`)

- 프로필 편집 (이름·표시 이름)
- 시험 정보 (시험일·목표 지역 교육청)
- 테마 (라이트/다크/시스템 자동)
- 알림 (준비 중)

## 4. 글로벌 UI 요소 (어디서나 보이는 것들)

### 4.1 좌측 사이드바 (`AppSidebar`)

- **데스크탑 (lg ≥ 1024px)**: 188px 폭, 항상 표시. 토글 가능.
- **모바일 (lg 미만)**: drawer 형태, 햄버거 버튼으로 열고 닫음. 배경 backdrop 클릭 또는 ESC 키로 닫힘.
- 상단: Fitly 로고 + X(닫기) 버튼
- 중단: 6개 메뉴 + D-day 칩 (시험일 등록 시 표시)
- 하단: 설정 + 로그아웃

**다크 모드 가시성** (2026-05-16 수정 완료): 다크 모드에서 비활성 메뉴 글자가 대시보드 "안녕하세요"와 동일한 밝은 cream 색으로 정합됩니다.

### 4.2 상단 헤더 (`PageHeader` / `DashboardHeader`)

- 모든 페이지 상단에 sticky로 고정
- 좌측: 페이지 제목 + 부제 (`PageHeader`) 또는 인사말 + 펀치라인 (`DashboardHeader`)
- 우측: 페이지별 액션 버튼 (대시보드는 "오늘 학습 시작" CTA + 알림 + **다크 모드 토글** + 프로필 아이콘)

### 4.3 팟캐스트 미니 플레이어 (`PodcastMiniPlayer`) — **세세한 디테일**

> **사용자 명시 요청 — 영상에 반드시 보여줘야 할 디테일입니다.**

**팟캐스트가 재생 중이면, 다른 페이지로 이동해도 화면 하단에 작은 오디오 플레이어가 계속 떠 있습니다.**

- 위치: 화면 **하단 sticky**, z-index 우선
- 높이: 76px (`MINI_PLAYER_H_PX`)
- 데스크탑(lg+): 사이드바 폭(188px)만큼 좌측 오프셋 — 사이드바와 겹치지 않음
- 구성: 에피소드 썸네일 · 제목 · 재생/일시정지 · 10초 rewind/fast-forward · 진행 막대 · 시간 · 닫기(X)
- 활성 에피소드가 있을 때만 노출 — 없으면 완전히 숨김
- 본문(main) 의 `padding-bottom` 이 미니 플레이어 높이만큼 자동 보정 — 콘텐츠가 가려지지 않음

**영상에서 보여줄 흐름** (강력 추천):
1. 팟캐스트 페이지에서 에피소드 재생 시작
2. 사이드바에서 **대시보드** 또는 **풀이 트랙** 메뉴 클릭
3. 페이지가 전환되어도 **하단 플레이어가 그대로 살아있고 오디오는 끊기지 않음**
4. 풀이 트랙에서 학습하면서 동시에 오디오 청취 — *"이동 중에도 학습"* 가치 시각화

이 한 장면이 **"진짜 PWA구나"** 를 가장 명확히 전달합니다. 시연 영상에 꼭 넣어주세요.

### 4.4 다크 모드 (`ThemeToggle`)

- 헤더 우측 상단 토글 (해/달 아이콘)
- 라이트 (cream 배경) / 다크 (deep warm ink 배경) / 시스템 자동 — 3 상태
- 사이드바·헤더·카드·차트·코드블록 모두 일관 적용

> **영상 멘트**: "토글 1회로 다크 모드 전환. 사이드바·차트까지 일관 톤."

### 4.5 모바일 PWA

- **홈 화면에 추가 가능** (manifest.json + 아이콘 셋)
- **오프라인 캐싱** (`public/sw.js` 서비스 워커) — 정적 자원·이전 페이지 캐시
- 모바일 햄버거 drawer
- iOS·Android 모두 PWA 설치 가능

> **영상 멘트**: "PWA로 설치하면 별도 앱처럼 홈 화면에서 바로 실행, 오프라인에서도 진입 가능."

## 7. 자주 묻는 디테일 — 영상에서 놓치기 쉬운 것들

### Q. 팟캐스트 미니 플레이어가 정확히 어떻게 작동하나요?
- 팟캐스트 페이지에서 에피소드 재생 → 화면 하단 76px 높이 sticky bar 등장
- 다른 페이지(대시보드·풀이·기출분석 등) 이동 → **플레이어 그대로, 오디오 끊김 없음**
- 일시정지·재생 버튼, 10초 rewind/fast-forward, 진행 막대 시각화
- 닫기(X) 클릭 시 완전 종료 + 플레이어 사라짐
- 본문 콘텐츠의 하단 padding 이 76px만큼 자동 늘어나 가려지지 않음
- 데스크탑(lg+): 사이드바 188px 옆에 정렬 / 모바일: 전 폭 사용

### Q. SplitView가 풀이/오답 트랙에만 있는 건가요?
- 네. **풀이(quiz) + 오답(mistake)** 트랙만 SplitView (워크스페이스 트랙)
- **키워드 트랙**은 단일 컬럼 (정리 노트 중심)
- xl(≥1280px) 이상에서만 활성, 그 미만은 세로 stack

### Q. AI 학습 도우미 챗봇(AssistantFab)은 어디에 있나요?
- **풀이 트랙에서만** 등장 (오답·키워드에는 없음)
- 우측 *중앙* 에 fixed 작은 ✨ 원형 버튼 (48×48px)
- 클릭 시 380×540px 챗 박스 펼침 — 본문과 동시 상호작용 가능 (모달 X, backdrop X)
- 빠른 프롬프트 5종: "이 문제 해설 줘" / "키워드 정리" / "관련 개념" 등
- 카드 전환 시 자동으로 대화 history 로드/리셋

### Q. 자가 채점 4 버튼이 어디에 있나요?
- 풀이/오답 트랙: 좌측 "내 답안" 카드 안 (답안 본문 아래)
- 키워드 트랙: 정리 노트 아래 별도 카드
- 4 버튼: 다시(1·error) · 어려움(2·warning) · 좋음(3·evergreen) · 쉬움(4·info)
- 키보드 단축키 1·2·3·4 동시 작동

### Q. 오답 트랙은 어떻게 채워지나요?
- 풀이 트랙에서 **"다시" 또는 "어려움"** 평가 시 → user_card_state.mark 자동 갱신 → 오답 트랙 큐에 합류
- 별도 "오답으로 등록" 버튼 없음 (개념 단순화)

### Q. PDF 인쇄 파일명은 어떻게 떨어지나요?
- 오답노트: `Fitly 오답노트.pdf`
- 기출분석 메인: `Fitly 기출분석.pdf`
- 기출분석 시험지 상세: `Fitly 2024학년도 1교시.pdf` 식
- 모두 `title.absolute` 메타데이터로 단정 (2026-05-15 수정 완료)

### Q. 다크 모드 토글은 어디에 있나요?
- 대시보드 상단 헤더 우측, 알림 종 아이콘 옆 (해/달 아이콘)
- 토글 1회 → 라이트 ↔ 다크 ↔ 시스템 자동 순환

### Q. 사이드바를 닫을 수 있나요?
- 데스크탑(lg+): 사이드바 폭 188px 토글 (열기/접기). 본문이 자동으로 폭 보정.
- 모바일: drawer 형태, 햄버거 버튼 / ESC 키 / backdrop 클릭으로 닫힘
- 사이드바 상단 X 버튼으로도 닫힘

## 8. 영상 멘트 톤 가이드

브랜드 언어 — **이 단어들을 반복해서 사용**해 주세요:

- **적합도** (핵심 펀치라인)
- **약점** (페인 ①)
- **24년치 공개 기출** (데이터 자산)
- **AI 모범답안 / AI 분석** (페인 ② 해소)
- **이동 중에도 학습** (페인 ③ 해소)
- **정직성** (반-인강 미학)
- **한 장 안에서 끝남** (UX 우수성)

**피해야 할 단어**:
- ✗ D-day 한정·마감 임박·합격 보장 (정직성 원칙 위배)
- ✗ "최고의" / "최강의" (과대 카피)
- ✗ 합격 가능성·합격 컷 추정 (보유 안 함)

**톤 예시**:
- ✅ "내 약점이 어디인지, 한 줄로 보여드립니다."
- ✅ "본인 데이터 기반. 외부 추정 없음."
- ✅ "이동 중 1.5시간이 학습 시간이 됩니다."
- ✗ "이것만 있으면 합격 보장!"
- ✗ "지금 가입하면 50% 할인!"

## 9. 마지막 — 영상의 진짜 목적

영상 5분이 끝났을 때 심사위원이 머릿속에 남겨야 할 **세 가지**:
