## 2. 제품 정체성 (헌법 v3.0 제2조 갱신안)

> **fitly v3.0**: 한국 초등교사 임용시험(1차)을 준비하는 자를 위한 학습 플래너 PWA. 24년치 공개 기출을 영역·인지수준·문항형식·키워드로 태깅한 시드 위에서, 사용자는 ① 출제 트렌드(히트맵·토픽맵·로드맵)로 약점을 발견하고, ② 서술형 풀이·키워드·오답의 3 트랙 SRS로 반복 학습하며, ③ NotebookLM 스타일 자동 생성 팟캐스트로 이동중에도 청취 학습한다.

**펀치라인 (제4조 유지·재해석)**

> "임용은 열심히 하는 게 아니라, 맞게(Fit) 하는 게임입니다."
> *해설 v3.0: 내 시험일·내 약점 영역·내 망각 곡선에 맞게.*

## 3. 핵심 동선 (헌법 5/6/7조 정합)

```
가입 (1탭)
  → 시험일 입력 (2탭)
  → 대시보드 (3탭, 즉시 첫 가치)
     ├─ D-day 카운트다운
     ├─ 오늘의 3 트랙 큐 (풀이 N장, 키워드 N장, 오답 N장)
     ├─ 24년치 영역별 히트맵 미리보기 (마법 지점 1)
     └─ 오늘의 추천 팟캐스트 1편 (자동 생성, 마법 지점 2)
```

- **첫 5초 가치 (제6조)**: 시험일 입력 직후 24년치 출제 분포 즉시 노출, SRS 큐 자동 채워짐
- **진입장벽 (제5조)**: 가입 + 시험일 = 2탭. 첫 자료 업로드는 선택(시드로 충족)
- **마법 지점 (제7조)**: ① 시드 분석 즉시 노출, ② 팟캐스트 자동 생성

## 5. 데이터 모델 (Drizzle 스키마 골격)

### 5.1 시드 (관리자 1회 입력, 모든 사용자 공용)

```ts
exam_papers {
  id, year, session ('논술' | 'A' | 'B'),
  pdf_url, source_url,
  verified boolean
}

exam_items {
  id, paper_id (FK), item_no,
  stem_md,                       // 문제 본문
  points number,
  format ('객관식' | '단답형' | '서술형' | '논술형'),
  domains text[],                // 11과목 + 교육학
  bloom ('기억' | '이해' | '적용' | '분석' | '평가' | '창작'),
  keywords text[],
  answer_key_md,                 // AI 생성 모범답안 (서술형만)
  verified boolean
}
```

### 5.2 카드 (시드에서 자동 파생, 트랙 분리)

```ts
cards {
  id,
  type ('quiz' | 'keyword' | 'mistake'),
  source_item_id (FK exam_items, nullable for mistake),
  user_id (NULL for shared seed cards, set for mistake/keyword pinned),
  front_md, back_md,
  fsrs_state                     // ts-fsrs state per (user_id, card_id)
}

user_card_log {
  id, user_id, card_id, grade ('again'|'hard'|'good'|'easy'),
  reviewed_at
}
```

- 풀이 카드(`type='quiz'`) — 2014~2026 서술형만 ≈ **585장**
- 키워드 카드(`type='keyword'`) — 2002~2026 풀스코프, 동일 키워드 dedup, ≈수천장
- 오답 카드(`type='mistake'`) — 사용자별, 풀이 again/hard 자동 합류

### 5.3 사용자 답안·청취 기록

```ts
user_attempts {
  id, user_id, item_id (FK),
  answer_md,                     // 사용자 작성 답
  self_grade ('again'|'hard'|'good'|'easy'),
  attempted_at
}

podcast_episodes {
  id,
  scope ('shared' | 'user'),     // shared = 시드 영역별 자동 생성
  user_id (nullable),
  theme,                          // ex: "2024 교직논술 / 2022 개정 교육과정"
  script_json,                   // multi-speaker dialogue
  audio_url,
  duration_sec,
  generated_at,
  verified boolean
}

podcast_progress {
  id, user_id, episode_id, current_sec, completed_at
}

users {
  id, email, exam_date,
  pace_mode ('light' | 'standard' | 'intensive')
}
```

## 6. 진척도 공식 (헌법 제9조 v3.0 갱신안)

```
Progress = (풀이 마스터율 × 0.5) + (키워드 마스터율 × 0.2) + (학습 일관성 × 0.3)
         (0~100 정규화)

풀이 마스터율    = FSRS state ≥ Review 인 quiz 카드 수 / 전체 quiz 카드 수
키워드 마스터율  = FSRS state ≥ Review 인 keyword 카드 수 / 전체 keyword 카드 수
학습 일관성     = 최근 14일 학습한 일수 / 14
```

### v2.0 → v3.0 변경점

| 항목 | v2.0 (편입 영어) | v3.0 (초등 임용) | 근거 |
|------|----------------|----------------|------|
| 1차 변수 | 어휘 마스터율 0.4 | **풀이 마스터율 0.5** | 임용 학습의 무게중심 = 서술 풀이 |
| 2차 변수 | 오답 정복률 0.3 | **키워드 마스터율 0.2** | 오답은 풀이 again/hard로 풀이 마스터율에 자연 흡수, 중복 회피 |
| 3차 변수 | 학습 일관성 0.3 | **학습 일관성 0.3** | 그대로 (도파민·성실성 트리거) |

> 제10조 금지어("합격 가능성", "학교 핏" 등) 그대로 유지. UI 라벨은 "학습 진척도 N/100 (본인 누적)"만 사용.

## 7. 시드 파이프라인 (1주일 1회성, 매년 1~2일 갱신)

```
[1] PDF 메타 정규화                ─ 파일명 → {year, session, pdf_url}
                                      ↓
[2] PDF → 텍스트 추출              ─ unpdf + Gemini OCR (이미지·표 보존)
                                      ↓
[3] 문항 분리                      ─ Gemini로 item_no/stem/points 추출
                                      ↓                                       ↓
[4-A] 풀이 시드 (2014~2026)         [4-B] 키워드 시드 (2002~2026)
  서술형만 필터링                     전 기간 객관식 포함
  Gemini로 영역/Bloom/형식 태깅       Gemini로 키워드만 태깅 (3~5개/문항)
  Gemini로 모범답안 생성             (모범답안·채점 X)
                                      ↓
[5] 사람 검토 (verified=true 토글)
                                      ↓
[6] 자동 카드 파생
  cards.type='quiz'    ← 풀이 시드 (≈585장)
  cards.type='keyword' ← 키워드 시드 (dedup 후, ≈수천장)
```

**검증 미완료 노출**: 검토 전 문항·모범답안은 UI에서 `verified: false` 배지(임용고시 레이아웃 이미지 6 그대로). 헌법 제3조의2 정직성·제18조의2 라벨 의무 정합.

## 8. 팟캐스트 (NotebookLM 스타일 자동 생성)

### 8.1 동선

```
사용자 진입점:
  - 대시보드의 "오늘의 추천 1편"
  - 팟캐스트 페이지에서 직접 선택

생성 단위:
  shared scope     — 시드 자동 생성 (영역×연도, 2명 화자 대화체)
  user scope       — 사용자가 자기 약점 영역 선택 시 즉석 생성

화자: 2인 한국어 (Gemini multi-speaker TTS)
길이: 5~12분 표준
청취 진척: podcast_progress 로 재개 가능
```

### 8.2 콘텐츠 출처 정직성

- 시드 기반 팟캐스트는 **공개 기출 + AI 생성 해설** 조합 → `verified: false` 배지로 노출
- 헌법 제18조의2(AI 생성 라벨 의무), 제27조(공시 출처 한정) 정합
- 팟캐스트 본문에 "이 팟캐스트는 AI 생성이며 공식 해설이 아님" 안내 의무

## 11. MVP 단계 계획

### Phase 1 (MVP — 첫 출시)

7페이지 + 시드 백엔드. 빌드 순서:

1. 시드 파이프라인 (`scripts/seed/*`): PDF → text → items → cards
2. 기출분석 페이지 (히트맵 3종 / 토픽맵 / 로드맵)
3. 풀이 페이지 (연도×교시 → 문항 → 자가 채점)
4. 학습 페이지 (3 트랙 SRS 큐, ts-fsrs 통합)
5. 팟캐스트 페이지 (Gemini TTS 통합, 라이브러리 + 청취 진척)
6. 대시보드 (위 데이터 집계 + D-day)
7. 학습계획 + 마이 페이지 (간이)

### Phase 2 (확장)

8. 오답 페이지 (풀이 again/hard 자동 합류 뷰)
9. 학습분석 (개인 진척도 시계열·영역별 마스터)
10. 자료 페이지 (사용자 PDF 업로드 + AI 카드 자동 추출)
