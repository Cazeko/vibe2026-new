## 1. 문제 (60초, 슬라이드 2장)

### Slide 1-1 — 배경 (25초)

```
한국 초등교사 임용 1차 시험

┌──────────────┬──────────────┬──────────────┐
│  응시자       │   시험 횟수    │  공개 기출    │
│  연 4만+      │  1년 1회      │  24년치       │
│              │  (11월 말)    │  2002~2026   │
└──────────────┴──────────────┴──────────────┘

기존 시장
  박문각·해커스 인강 패스 — 80~150만원/시즌
  종로학원 모의고사 — 90,000원/3회

문제: 24년치 데이터는 공개되어 있는데,
      *내 약점이 어디인지* 알려주는 도구가 없습니다.
```

**발표 대본 (25초)**

> "한국 초등교사 임용 1차는 연 4만 명이 응시하는 1년 1회 시험입니다.
> 기출은 24년치가 모두 공개돼 있고, 시중에는 박문각·해커스 같은 80~150만원짜리 패스 인강이 있습니다.
> 그런데 진짜 문제는, *24년치 데이터가 있어도 내 약점이 어디인지 알려주는 도구가 없다는 것* 입니다."

## 2. 솔루션 (60초, 슬라이드 2장)

### Slide 2-1 — 핵심 가치 (25초)

```
                  Fitly = "Fit (맞게) 학습"

┌─────────────────────────────────────────────────┐
│                                                 │
│   내 시험일·내 약점·내 망각 곡선에                 │
│              "맞게(Fit)" 학습합니다.              │
│                                                 │
└─────────────────────────────────────────────────┘

세 개의 마법 지점
  ① 24년치 기출 즉시 분석     →  약점 시각화
  ② AI 모범답안 + 5탭 분석    →  자가 채점 명확
  ③ NotebookLM 스타일 팟캐스트 →  이동 시간 학습
```

**발표 대본 (25초)**

> "Fitly 의 핵심 가치는 단 한 줄입니다 — *내 시험일, 내 약점, 내 망각 곡선에 '맞게(Fit)' 학습한다*.
> 그리고 이 가치를 세 개의 마법 지점으로 풀어냅니다.
> 첫째, 24년치 기출 즉시 분석. 둘째, AI 모범답안 + 5탭 분석. 셋째, 자동 생성 팟캐스트."

### Slide 2-2 — 페인-가치-기능 매핑 (35초)

```
페인 ────────── 핵심 가치 ────────── Fitly 기능

①  약점 진단     →   기출 즉시 분석    →   기출분석 페이지
   부재                                    (히트맵·토픽맵·
                                          로드맵)

②  자가 채점     →   AI 채점 + 분석    →   풀이 트랙 + 5탭
   모호                                    (총평/키워드/diff/
                                          모범답안/이력)

③  이동 시간     →   자동 팟캐스트     →   팟캐스트 페이지
   비효율                                  (Gemini multi-
                                          speaker TTS)

보조 기능
  · 3 트랙 SRS (풀이/키워드/오답) — ts-fsrs 통합
  · 학습 워크스페이스 (마크/필터/타이머/소문항 분리)
  · AI 학습 도우미 챗봇 (FAB)
  · 학습 계획 + 학습 분석 + 마이 페이지
```

**발표 대본 (35초)**

> "이 세 가지 페인을 세 개의 기능에 1:1로 매핑했습니다.
> 약점 진단 부재 → 기출분석 페이지에서 24년치 데이터로 영역별 약점 시각화.
> 자가 채점 모호 → 풀이 트랙 + 5탭 분석에서 AI 모범답안과 차이를 명확히.
> 이동 시간 비효율 → 자동 생성 팟캐스트로 통학 시간을 학습 시간으로.
> 보조 기능으로 3 트랙 SRS, 학습 워크스페이스 풀, AI 챗봇이 함께 들어갑니다.
> 이제 실제로 작동하는 모습을 보여드리겠습니다."

## 7. 회고·소회 (30초, 슬라이드 1장)

### Slide 7-1 — AI 협업 회고 + 극복점 1개

```
14일 동안

  225 commits  · 56 PR · 6 단계 PR 시리즈
  헌법 v3.0 → v3.6.4 (4 계층 분리)

극복점 — AI silent fallback 회귀 (PR #71)

  증상     채점 후 AI 총평·키워드·diff 모두
          "분석 결과가 비어 있습니다" 표시

  원인     LLM JSON 파싱 실패 시 빈 배열로
          *조용히* fallback → UI 가 "성공인데
          빈 답안" 으로 잘못 안내

  해결     파싱 실패는 명시 throw + 진단 로그
          (head/tail/finishReason) + 출력 토큰
          2048 → 4096 격상

배움  ──  AI 출력 처리에서 fallback 은
          *명시 에러로 surface*, silent 절대 금지.
```

**발표 대본 (30초)**

> "마지막으로, 14일 AI 협업 회고를 짧게.
> 225 commits, 56 PR, 헌법 v3.6.4 까지 진화시켰습니다.
> 가장 인상 깊은 극복점은 — AI 분석 결과가 빈 화면으로 표시되는 회귀였습니다.
> LLM JSON 파싱이 실패하면 코드가 *조용히* 빈 배열로 fallback 하고 있었습니다.
> 사용자는 '내 답안이 부족하구나' 오해하지만, 실제로는 LLM 응답이 토큰 한도에 잘렸던 것입니다.
> 여기서 배운 원칙 — *AI 출력 처리에서 fallback 은 명시 에러로 surface 해야 한다, silent 는 사용자에게 정직하지 않다*. 우리 헌법 §3의2 와 정합하는 원칙으로 코드에 박았습니다.
> 감사합니다."
