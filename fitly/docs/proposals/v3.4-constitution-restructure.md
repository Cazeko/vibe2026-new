### 2.3 개정 사유

1. **헌법층 비대화의 진단** — v3.3 시점 헌법은 581줄 55조항. R004 H2의 정확한 반례. 신규 진입자(Claude·Gemini·사람)가 매 세션 700줄 로드 + 인지 부하 누적.
2. **충돌 해소 알고리즘 자동화** — 4계층 위계 명시 시 *신법·특별법 우선* 원칙으로 충돌 자동 해소 (R004 H1).
3. **자가집행성 강화** — 현 제38조는 *위계를 언급하나 물리 분리 없음* → 위계가 *논리적*이지 *물리적*이지 않음. R004 H4의 자기집행성 부족 → AI 일관성 저하 위험.
4. **토큰 비용 절감** — Claude Code가 매 세션 CLAUDE.md 전체 로드(~13K 토큰). 분리 후 헌법층(~50줄, ~1K 토큰)만 로드 + 필요 시 참조 → 매 세션 12K 토큰 절감.
5. **개정 안전성 향상** — 현재 v3.x 활발 변경 중. 분리 시 *영향 받는 계층만 수정* 가능 → 다른 계층 안정성 보존.

**코드·기능**:
- 영향 없음 (코드는 *헌법 인용*만 변경, 실제 로직 동일)
- DESIGN.md, drizzle schema, src/ 영향 없음

**작업 비용**:
- 4~6시간 (조항 위계 분류 + 16개 파일 작성 + frontmatter + 검증)
- 사용자 검토 시간 추가 (각 조항이 *맞는 계층에 있는가* 사용자 확인)

**위험**:
- 조항을 잘못된 계층에 배치 시 위계 작동 실패 → 사용자 검토로 1차 보호
- 분리 작업 중 다른 worktree(`feat/v3-seed-pipeline`)와 충돌 → 작업 시점 분리 필요
- v3.x 빈번 개정 중 분리 → 분리 직후 v3.5 신규 개정 시 어디 계층에 들어갈지 판단 필요 (단, 본 발의 채택 시 *위계 분류 자체가 발의 절차의 일부*가 됨)

**검증 의무 (제24조의3 정합)**:
- 본 분리는 *코드 변경 없음* → `npm run build` 의무 아님
- 다만 `.claude/` 디렉토리 구조 변경 → Claude Code의 헌법 로드 검증 필요 (1세션 dry-run)
