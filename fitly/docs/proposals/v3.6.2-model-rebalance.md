## 1. 발의 배경

### 1.1 v3.6.1 매트릭스의 환경/리전 미가용 사고 (LlmFailed)

PR #44 (2026-05-14 머지) 채택 시점에 매트릭스 A 의 Flash 행이 `gemini-3.0-flash` 로 명시되었고, `env.modelFlash` fallback 도 동기화하였다. 그러나 운영 환경에서 `gemini-3.0-flash` 호출 시 `generateContent` 가 throw 하며 LlmFailed 가 발생하였다 (모델 ID 가 환경/리전별 미가용 가능성).

본 발의로 fallback ID 를 *가용성이 검증된 안정 세대* (`gemini-2.5-pro` / `gemini-2.5-flash`) 로 복원하되, 환경변수(`GEMINI_MODEL_PRO`, `GEMINI_MODEL_FLASH`)를 통한 *런타임 신모델 업그레이드 경로* 는 §18-5 정합으로 보존한다.

### 1.2 주인님 직접 발화 (2026-05-14) — 기능별 7개 매핑 명세

다음 매핑이 명시되었다.

| 기능 | 티어 |
|---|---|
| 실시간 챗봇 | Flash |
| 힌트 생성 | Flash |
| 암기법 생성 | Flash |
| 키워드 추출 | Flash |
| 서술형 채점 | Pro |
| 첨삭 | Pro |
| 모범답안 생성 | Pro |

이 매핑은 *기능 단위* 분류이며, v3.1 매트릭스의 "추론 깊이 ↔ 가성비" 차등 정책의 *세부 적용* 으로 해석된다. 다만 매트릭스에 명시적으로 등재되어 있지 않은 항목(힌트/암기법/모범답안 생성)이 있어, 본 발의로 매트릭스 A 에 명시 등재한다 (§18-6 "매트릭스를 변경하려면 헌법 개정 절차를 거친다" 정합).

### 2.3 개정 사유

1. **안정 세대 복원** — `gemini-3.0-flash` 미가용 LlmFailed 사고 정합. fallback 안정성 회복 + 환경변수 런타임 업그레이드 경로 보존
2. **주인님 7개 매핑 명세 등재** — 기능별 차등 매핑을 매트릭스에 명시. 향후 힌트/암기법/모범답안 생성 기능 추가 시 매트릭스 정합 자동 보장
3. **추론 깊이 ↔ 빠른 응답 분리 강화** — 채점/첨삭(정성 추론) Pro, 챗봇/힌트(빠른 응답) Flash. v3.1 매트릭스 차등 정책 세부 적용
