## 1. 배경 (왜 개정하는가)

PR #22 (v3.5.3 omnibus) 머지 후 미룬 항목 4 트랙이 식별되었다 (`docs/proposals/2026-05-12-external-review-deferred.md`). 주관자가 본 4 트랙 전부 빠짐없이 진행하라는 명시 발화를 주셨고, 그중 Track 3 (과목 컬러 코딩) 은 DESIGN.md §4.4 "라벨+아이콘+좌측 보더 3축 only" 원칙과 충돌한다. Track 1 (Split View, Global Mini-player) 은 §16 신규 기능 추가 금지 와 결을 같이 검토할 필요가 있다.

## 2. 개정 조항 (5요소 — 제42조 정합)

### 2.1 DESIGN.md §4.4.1 신설 — 영역(domain) 분류 단서

**개정 전**: §4.4 시맨틱 컬러 운영. *"차별화는 (라벨+아이콘+좌측 보더 3px) 3축으로만 한다. 배경 채움이나 강한 밴드는 사용하지 아니한다."*

**개정 후 (§4.4.1 신설)**: 본 §4.4 의 "라벨+아이콘+좌측 보더 3축" 원칙 안에서, **좌측 보더의 *hue* 만은 영역(domain)·과목 분류 목적으로 다색 허용** 한다. SRS 4등급 시맨틱 (2026-05-07 §4.4 예외) 와 같은 *학습 인지 핵심* 단서로 인정한다.

**조건**:
- 배경 채움·텍스트 컬러는 변경하지 아니한다. 좌측 보더 3~4px 만 hue 차별화한다.
- hue 는 *낮은 채도 (S ≤ 35%)* + 본 페이지의 1% 종이 그레인 위에서 충돌하지 아니하는 영역(`--subject-*` 토큰) 안에서만 선택한다.
- §4.3 6 사용처 보호 — evergreen·gold 본연 hue 는 본 단서 토큰에 포함하지 아니한다.

**근거 조항**: §4.3 "한 번만 빛난다" 원칙과 충돌하지 아니한다 — 좌측 3px 보더는 본 메시지의 *카드 강조 빛* 이 아니라 *식별자* 다.

### 2.2 §16 단서 (v3.5.3) 명시 적용

Track 1.1 (Split View) · Track 1.2 (Global Mini-player) · Track 2.1 (Lazy Loading) · Track 2.2 (WCAG audit) · Track 3 (Subject color coding) — 모두 v3.5.3 §16 단서 *"외부 사용성 평가·접근성 점검·시각 위계 보강에 기반한 UI 개선은 기존 기능 다듬기로 분류한다 (라이트박스/zoom 컨트롤러·스플릿 뷰·전역 미니플레이어 등 기존 기능의 인터랙션 패턴 보완 포함)"* 정합으로 *기존 기능 다듬기* 분류.

### 2.3 §3.2 (정직성) — Track 4 보수적 NoChange 결정 문서화 의무 (신설)

외부 사용성 평가에서 제기된 항목 중 *본 PR 에서 적용하지 아니하는* 항목은 §3.2 정직성 정합으로 *결정 근거 + 재평가 조건* 을 문서로 기록한다. 본 의무는 외부 평가 ≥10 건 이상 단일 PR 처리 시에 적용된다.
