## 2부 — NoChange 3건 (결정 기록)

### NoChange 1 — #2.2 LNB·KPI vs dashboard-header subtitle 의 D-XX 중복

#### 외부 평가 의도
> *"LNB 하단에 'D-198 서울 시험 대비' 카드가 있는데, 메인 영역 상단 목표 카드에도 똑같은 내용이 있습니다. 시선 분산입니다. LNB 하단 카드는 빼거나 위젯 형태로 작게 축소하세요."*

#### 결정
**NoChange** — 현행 시각 분리 유지.

#### 근거
- 현황 점검:
  - `app-sidebar.tsx` LNB 하단 — D-XX 카드 *제거됨* (이전 PR 에서 정합)
  - `kpi-cards.tsx` Goal 카드 — `bg-evergreen border-evergreen text-cream` (유일한 evergreen 채움) + D-XX subtitle
  - `dashboard-header.tsx` subtitle — *"{지역} 시험까지, 오늘의 적합도 한 점 함께 짚어 드릴게요."* 동기부여 카피 (D-XX 숫자 미노출, 지역명만)
- 외부 평가가 지적한 *동일 정보 중복* 은 이미 *시각 위치 + 톤 분리* 로 해소.
- KPI 카드는 *수치 (D-XX)* 와 *행동 유도 (D-XX·2026학년도 1차)* 에 집중, header subtitle 은 *지역명 + 동기부여* 에 집중.

#### 재평가 조건
사용자 또는 baseline 3 명 이상의 베타 학습자가 *"같은 정보가 두 번 보인다"* 류 피드백 ≥3 건 누적 시 subtitle 에서 지역명 제거 또는 KPI Goal 카드 라벨 축소 검토.

### NoChange 2 — #5.6 설정 폼 max-width 600px

#### 외부 평가 의도
> *"인풋 필드의 세로 폭(Height)이 화면에 비해 너무 넓고 길게 늘어져 있습니다. Max-width를 600px 정도로 제한하여 중앙 혹은 좌측에 정돈된 폼 레이아웃을 만드세요."*

#### 결정
**NoChange** — 현행 `max-w-5xl` + grid 3col 유지.

#### 근거
- 현황 점검 (`src/app/(main)/settings/page.tsx`):
  - 페이지 wrapper — `max-w-5xl` (1024px)
  - grid 3col — 좌측 col-span-2 폼 카드 약 640px, 우측 col-span-1 테마/계정/정책 카드 약 320px
  - 좌측 카드 안 `<Input>` `<Select>` 폭 — 약 600px (외부 평가 의도 부합)
- 페이지 전체 폭이 아닌 *폼 카드 내부* 폭이 약 600px 이므로 평가 의도와 *실효 정합*.
- 단일 폼 max-width 제한은 우측 테마/계정/정책 카드 배치를 잃게 되어 시각 효율 손실.

#### 재평가 조건
1280px+ 와이드 데스크톱에서 *"폼이 한쪽으로 치우쳐 보인다"* 또는 *"좌우 컬럼이 어색하다"* 피드백 ≥3 건 누적 시, max-w-2xl (672px) 단일 컬럼 또는 mx-auto 중앙 정렬 검토.

### NoChange 3 — #3.9 오답 트랙 Empty State Primary Button (evergreen 채움 X)

#### 외부 평가 의도
> *"오답 트랙의 '학습 계획으로' 텍스트는 전혀 버튼처럼 생기지 않았습니다. 명확한 Primary Button 형태(초록색 배경에 흰 텍스트)로 만들어 다음 액션을 확실히 유도하세요."*

#### 결정
**NoChange** — `outline` 변형 유지 (evergreen 채움 미적용).

#### 근거
- 현황 점검 (`src/app/(main)/study/[track]/page.tsx` `EmptyQueue`):
  - P1-04 (외부 리뷰 2026-05-12) — 종전 `ghost` → `outline` 으로 이미 한 단계 격상됨
  - `ArrowLeft` 아이콘 + "학습 계획으로 돌아가기" 시각 어포던스
  - 다른 트랙 due 가 있으면 상단에 *별도 강조 카드* (`border-l-evergreen` `bg-evergreen/5`) 로 우선 노출 — 진짜 Primary 행동은 별도 처리
- §4.3 evergreen 6 사용처 보호 — *학습 시작 CTA* 1곳에만 evergreen 채움 (KPI Goal · Dashboard "오늘 학습 시작" · 학습 계획 진입 CTA · AI 추천 · 활성 메뉴 · 진척도) 정합. Empty 의 "돌아가기" 는 *복귀* 행동으로 evergreen 자격 외.
- §16 단서 (v3.5.3) 정합 — 외부 평가는 *기존 기능 다듬기* 분류, 다만 §4.3 액센트 보호가 우선 (§38 위계 — DESIGN.md 법률층 동급 > 시행규칙).

#### 재평가 조건
- §4.3 evergreen 6 사용처 정책 자체가 개정되거나
- 사용자(주관자) 명시 발화로 *"돌아가기 CTA 를 evergreen 으로 격상"* 지시 발생
- 이 경우 §4.3 발의안 + 별도 PR 로 단행
