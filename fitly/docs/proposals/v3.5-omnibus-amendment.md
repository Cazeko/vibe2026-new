## 1. 발의 배경

v3.4 4계층 분리(2026-05-06) 직후 다음의 *구조적 누락*과 *시드 운영 모호점*이 동시에 관찰되었다:

1. **헌법층 누락 (Claude 측 자기 점검)**
   - 제1조(명칭) — Fitly의 정식 명칭이 헌법층 어디에도 정의되지 아니함. 제2조에서 "Fitly"가 *언급*되지만 *정의*되지 아니함.
   - 제3조(사명) — 정직성 원칙(제3조의2)은 있으나 *사명문(mission statement)* 부재.
   - 부칙(시행·경과 규정) — 시행일·경과 규정·길이 상한이 헌법 본문에 부재 (CHANGELOG에만 일부 산재).

2. **시드 운영 모호점 (다른 세션 Opus 검토 결과)**
   - 객관식 시대(2002~2013) *연도 범위* 모호 (CHANGELOG v3.2에 표현은 있으나 *시기 경계*가 헌법 본문에 명문화 X).
   - `exam_items.points` 타입 — schema는 `integer`로 작성되어 있으나 헌법 본문에 *number 의무*가 명시 X.
   - 2001학년도 PDF 폰트 손상 — `unpdf` 실패 시 백업 경로가 *연도 특정*으로 명문화 X.
   - LLM over-tagging — 태그 상한·우선순위가 어떤 법률·시행령에도 부재.

본 발의는 이 5건 + 헌법 누락 3건을 묶어 *omnibus 1회 처리*로 정합성을 회복한다.

### 2.2 제안 조항

#### A. 제1조 (명칭) [v3.5 신설 — 헌법층 `.claude/CONSTITUTION.md`]

> 본 시스템의 정식 명칭은 **"Fitly"** 이며, 펀치라인(제4조)의 "맞게(Fit)"와 어원을 공유한다. 외부 노출(UI·홍보·문서·도메인)에서 "fitly", "FITLY" 등 변형은 *공식 채택 대안 표기*로만 허용한다. 본 명칭은 사용자(주관자) 명시 승인 없이 변경하지 아니한다.

#### B. 제3조 (사명) [v3.5 신설 — 헌법층]

> Fitly의 사명은 **"임용 학습의 정직한 자동화"** 이다. 사설 학원·인강·족보 의존 환경의 격차를 *공개 기출 시드 분석·역산 학습 플래닝·정답 4계층 검증*으로 완화한다. 합격 가능성 추정·지역 컷 추정 등 외부 비공개 데이터 의존 약속을 일체 하지 아니한다 (제3조의2 정합).

#### C. 부칙 (시행·경과) [v3.5 신설 — 헌법층 또는 별도 `.claude/BYLAWS.md`]

> 1. 본 헌법은 채택일부터 시행한다.
> 2. v3.4 4계층 분리는 2026-05-06부터 시행됨을 확인한다.
> 3. 헌법 개정 시 `.claude/CHANGELOG.md`에 즉시 반영한다 (제45조 정합).
> 4. **길이 상한 권고**: 헌법층 ≤ 80줄(R004 H2 강제), 법률층 각 ≤ 200줄, 시행령층 각 ≤ 150줄, 시행규칙층 무한 (R004 H3 정합).
> 5. v3.x 기간(임용 1차 D-90~D-0)은 *시행규칙층 무한 성장* 허용. 단 헌법층·법률층은 본 권고 상한 준수.

#### D. 제18조의3 (객관식 시대 데이터 운영 — 연도 범위 명문화) [v3.5 신설 — `.claude/laws/15_ai_models.md`]

> 1. **객관식 시대 = 2002학년도 ~ 2013학년도 12회분**으로 정의한다 (KICE 공식 시험 형식 변천 정합).
> 2. 객관식 시대 기출은 **풀이 카드(QuizCard) 출제 대상에서 제외**한다 (CHANGELOG v3.2 결정 명문화).
> 3. 객관식 시대 데이터는 **개념 정리 노트(KeywordCard)** 로 흡수하여 RAG·토픽맵·로드맵에 한해 활용한다.
> 4. 2014학년도부터는 **서술형 시대**로 분류하며, 풀이 출제 대상이 된다.
> 5. 2002학년도 이전(2001 이하) 자료는 시드에 포함하지 아니한다 (제F조 정합).

#### E. 제19조의2 (시드 schema points 타입) [v3.5 신설 — `.claude/laws/14_tech_stack.md`]

> 1. `exam_items.points` 컬럼은 **`integer` (number)** 타입으로 정의한다.
> 2. UI 표시에서 `0.5점` 등 소수 점수가 등장하는 경우 별도 컬럼(`points_fractional`) 또는 `basis_points (1/100)` 정수 변환을 사용하되, 본 헌법 채택 시점(v3.5)에는 *전 영역 정수 점수*만 시드에 포함됨을 확인한다 (KICE 임용 1차 채점 기준 정합).

#### F. 제35조의2 (시드 파이프라인 추가 위험·백업) [v3.5 신설 — `.claude/rules/33_failure_response.md`]

> 다음 추가 위험에 대한 백업 플랜을 명시한다:
>
> | 위험 | 백업 |
> |---|---|
> | 2001학년도 이전 PDF 폰트 손상 (한글 깨짐 / unpdf 실패) | 시드 대상에서 제외 (제18조의3 5항). 2002학년도부터 시드 시작 |
> | 특정 학년도 unpdf 부분 실패 | pdftocairo PNG → Gemini Vision OCR 폴백 (제18조 1항·B 매트릭스 정합) |
> | LLM 태그 폭주 (over-tagging) | 제25조의2 태그 상한 정책 적용 |

#### G. 제25조의2 (시드 LLM 태그 상한 정책) [v3.5 신설 — `.claude/laws/15_ai_models.md` 또는 시행규칙 `rules/35_seed_tagging.md` 신설]

> 시드 파이프라인 step03 (영역/Bloom/형식/키워드 태깅) 시:
>
> 1. **태그 상한 = 5개** (한 문항당)
> 2. **우선순위**: ① 영역(domains) → ② 인지수준(bloom) → ③ 문항형식(format) → ④ 키워드(keywords, 최대 2개) → (선택) ⑤ 단원
> 3. 태그 5개 초과 시 *낮은 우선순위 항목부터 제거*
> 4. system prompt에 본 규칙을 *명시 인용*한다 (헌법 근거 의무 — 제24조의2)

### 2.3 개정 사유

1. **헌법층 누락 보완**: 신규 진입자(인간·AI)가 헌법층 7원칙만으로 "Fitly가 무엇인가"의 *근본 정의*를 즉시 알 수 없는 R004 H2 반례. 명칭·사명·부칙은 *7±2 슬롯*에 충분히 들어가는 메타 정보.
2. **시드 운영 모호점 해소**: 객관식 시대 연도 경계가 헌법에 명시되지 않으면 step02 분류 단계에서 "2013학년도가 객관식인가" 판단 모호 → 데이터 누락·중복 위험.
3. **points 타입 명문화**: schema는 이미 `integer`이지만 헌법 부재 → 향후 마이그레이션·외부 ETL 시 충돌 위험.
4. **2001 PDF 처리**: 운영자(주인님)이 step01에서 "왜 2001만 OCR 실패가 잦은가" 의문 → 헌법에 *시드 제외 명문화* 가 정합.
5. **over-tagging 정책**: LLM 태그가 10+개로 폭주하면 토픽맵·기출분석 페이지 시각적 혼란 + RAG 정확도 저하.
