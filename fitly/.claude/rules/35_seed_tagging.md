# 시행규칙 35 — 시드 LLM 태그 상한 정책

> 본 시행규칙은 헌법 v3.5 제25조의2의 본문이다 (2026-05-06 신설).
> 시드 파이프라인(`docs/harness/seed-llm-prompts.md`) step03 (영역/Bloom/형식/키워드 태깅) 단계에 적용된다.

## 제25조의2 (시드 LLM 태그 상한 정책) [v3.5 신설]

1. **태그 상한 = 5개** (한 문항당 — 영역·인지수준·형식·키워드 통합).
2. **우선순위** (높음 → 낮음):
   1. 영역(`domains`) — 11과목 + 교육학 (필수)
   2. 인지수준(`bloom`) — 기억·이해·적용·분석·평가·창작 (필수)
   3. 문항형식(`format`) — 객관식·단답형·서술형·논술형 (필수)
   4. 키워드(`keywords`) — 최대 2개 (보조)
   5. (선택) 단원 — 영역 하위 (있으면 키워드 슬롯 1개와 교환)
3. 태그 5개 초과 시 *낮은 우선순위 항목부터 제거*한다.
4. system prompt에 본 규칙을 *명시 인용* 한다 (헌법 제24조의2 — 헌법 근거 의무).
5. 운영자가 5개를 초과해서 보존이 필요하다고 판단하는 경우, 사용자(주관자) 승인 후 본 시행규칙을 개정한다 (제42조 정합).

## 적용 시점

- step03 (분류·태깅) system prompt에 본 조항 인용 추가.
- step03 산출 JSON validator에서 태그 카운트 검증 (≤ 5).
- 검증 실패 시 LLM에 재요청 또는 운영자 검토 큐로 이동.
