# 시행규칙 33 — 위험·백업·장애 응답

> 본 시행규칙은 헌법 v3.3 제10장 (제35~37조)의 본문이다.

## 제35조 (백업 플랜) [v3.0 갱신]

다음 위험 발생 시 즉시 백업안으로 우회한다.

| 위험 | 백업 |
|---|---|
| Gemini PDF OCR 실패 (표·이미지 보존 어려움) | 운영자 텍스트 입력 + 부분 자동화 |
| Gemini multi-speaker TTS 가용성 변동 | 단일 화자 + UI 안내 (Phase 2 외부 임베드 검토 — 헌법 개정 필요) |
| AI 모범답안 품질 저하 | `verified: false` 배지 + 사용자 신고 채널 + 운영자 검토 큐 |
| Supabase Free 한도 초과 | Pro($25/월) 즉시 업그레이드 |
| ts-fsrs 통합 버그 | Leitner 5박스로 폴백 |
| 시드 검토 부담 폭주 | Phase 1은 풀이 카드 검토 우선, 키워드는 자동 신뢰 |

## 제36조 (시간 부족 시 우선순위 — v3.0)

3주 일정이 위협받을 경우 다음 순서로 자원을 보존한다.

1. **시드 파이프라인 (PDF → 텍스트 → 태깅 → 카드)** — 모든 기능의 토대, 절대 cut 금지
2. **기출분석 페이지 (히트맵 3종 + 토픽맵 + 로드맵)** — 첫 가치 마법 지점 1
3. **풀이 페이지 + 자가 채점** — 학습 본업
4. **학습 페이지 (3 트랙 SRS 큐)** — ts-fsrs 통합
5. **팟캐스트** — 첫 가치 마법 지점 2 (TTS 통합 어려우면 Phase 2로)
6. **대시보드 + 학습계획 + 마이** — 통합·메타
7. 그 외 (오답·학습분석) — Phase 2 보류 [v3.0.1 — 자료 cut]

## 제37조 (장애 시 사용자 보고)

빌드 실패·API 키 누락·데이터 결측 등 차단 요소가 발생하면 사용자에게 **즉시 보고**한다. 사용자에게 알리지 않고 임의 우회를 시도하지 아니한다.
