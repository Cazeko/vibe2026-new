# 법률 16 — 디자인 시스템

> 본 법률은 헌법 v3.3 제16조의2의 본문이다 [v2.1 신설].

## 제16조의2 (디자인 시스템 — DESIGN.md 우선)

1. 모든 시각·UI 결정(폰트·색·여백·레이아웃·모션·컴포넌트 양식)은 `fitly/DESIGN.md` 의 **단일 소스 오브 트루스(Single Source of Truth)** 를 따른다.
2. 폰트·색·여백·레이아웃·모션의 어떠한 변경도 **사용자(주관자) 명시 승인** 없이 이루어지지 아니한다. 본 헌법 제16조의2의 변경은 헌법 개정에 준한다.
3. 메모러블 앵커는 **"공책 가로 괘선 + 세리프 헤드라인 + 학습 진척도만 컬러로 빛난다"** 이며, 이후 모든 시각 결정은 본 앵커를 *돕는가 거스르는가* 로 판단한다.
4. 액센트 컬러 `#1F5C4A` 는 *오직* (a) Progress 게이지 (b) 1차 액션 CTA (c) +N% 델타 (d) 활성 사이드바 항목 (e) AI 추천 카드 강조 — 5곳에만 등장한다. 그 외 컴포넌트 사용은 절대 금지한다.
5. AI 도우미는 코드 작성 전 `fitly/DESIGN.md` 를 우선 참조하며, 위반된 코드를 발견 시 즉시 수정 제안한다. QA 단계에서 본 헌법과 DESIGN.md 의 일치성을 확인한다.
6. 디자인 시스템 미리보기 HTML 은 `~/.gstack/projects/Cazeko-vibe2026-new/designs/design-system-20260505/preview.html` 에서 라이트·다크 양 모드 렌더링으로 검증된다.
7. 절대 금지 폰트(Inter, Roboto, Arial, Helvetica, Open Sans, Lato, Montserrat, Poppins, Space Grotesk, Comic Sans, Papyrus 등)는 본 시스템의 어떠한 사이즈·역할로도 등장하지 아니한다 (DESIGN.md 제3장 정합).
