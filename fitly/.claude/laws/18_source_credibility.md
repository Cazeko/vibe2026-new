# 법률 18 — 정답·해설 출처 4계층 모델

> 본 법률은 헌법 v3.3 제30조의2의 본문이다.

## 제30조의2 (정답·해설의 출처 4계층 모델)

모든 카드의 정답·해설은 다음 4출처 중 하나로 명시 라벨링된다. 라벨 없는 정답·해설은 노출하지 아니한다.

| 라벨 | 영문 enum | 의미 | UI 배지 |
|---|---|---|---|
| 공식 | `official` | 학교 공시 정답 또는 검증 가능한 1차 출처 | "공식" (회색) |
| AI 추정 | `ai_estimate` | Gemini가 본문으로부터 추정한 답·해설 | "검증 필요" (앰버) |
| 자가 정정 | `user_self_corrected` | **단일 사용자(본인)** 가 직접 정정한 답·해설 | "내가 정정" (스카이) |
| 크라우드 검증 | `crowd_verified` | **2명 이상의 서로 다른 사용자가 동일한 답을 제출**하여 검증된 답 | "검증됨" (인디고) |

1. 단계적 결합 원칙 — 시점에 따라 다음 순서로 결합한다.
   - 단계 1: AI 추정으로 즉시 노출 (콜드 스타트)
   - 단계 2: 사용자 본인이 자기 카드를 정정한 경우 `user_self_corrected`로 변경 (단일 의견)
   - 단계 3: 학교별 공식 정답지가 확보되는 대로 `official` 승격
   - 단계 4: 동일 카드에 **2명 이상의 서로 다른 사용자가 동일 답을 제출**한 경우 `crowd_verified` 승격
2. `user_self_corrected`는 그 자체로 객관적 검증을 의미하지 아니하며, 본인의 자가 신뢰만을 표상한다. UI는 이 라벨을 "검증됨"과 시각적으로 구분되도록 표시해야 한다.
3. 라벨 변경 이력은 카드별 메타에 기록한다.
4. 발표·UI·홍보에서도 이 4계층을 신뢰성 방어 논리로 일관되게 사용한다 (제12조 정합).
