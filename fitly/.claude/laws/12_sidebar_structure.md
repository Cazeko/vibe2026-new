# 법률 12 — 사이드바 구조 + 카드 분류 + 팟캐스트

> 본 법률은 헌법 v3.3 제4장 일부 (제13조·13조의2·13조의3)의 본문이다.

## 제13조 (사이드바 7+1+2 구조 — 변경 금지) [v3.0 전면 개정]

사용자 인터페이스는 **태블릿 가로(1024×768 기준) PWA 데시보드**로 한다. 좌측 고정 사이드바 + 우측 콘텐츠 그리드 구성을 표준으로 한다. 사이드바 메뉴는 다음 **Phase 1: 7개 + 신규 1개 (팟캐스트) + 시스템 2개**로 한정한다. Phase 2에서 3개(오답·학습분석·자료)를 추가한다 (신규 메뉴의 추가는 헌법 개정에 준한다).

**Phase 1 메인 메뉴 7 + 신규 1**:
1. **대시보드** (`/dashboard`) — 인사말 / KPI 4 (D-day·Progress·학습 시간·연속) / 24년치 영역 히트맵 미리보기 / 오늘의 3 트랙 큐 / 추천 팟캐스트 1편
2. **기출 분석** (`/exam-analysis`) — 4개 탭: ① 기출문제(연도×교시 색인) ② 분석(영역×연도 / 인지수준×연도 / 문항형식×연도 히트맵 3종) ③ 토픽맵(키워드 클라우드 + 반복 출제 키워드) ④ 로드맵(S/A/B/C 우선순위)
3. **풀이** (`/study/quiz`) — 연도×교시 → 문항별 답안 작성 + AI 모범답안 비교 + 자가 채점(FSRS grade)
4. **학습** (`/study`) — 3 트랙 SRS 큐: 풀이(QuizCard) / 키워드(KeywordCard) / 오답(MistakeCard)
5. **팟캐스트** (`/podcast`) — NotebookLM 스타일 자동 생성 라이브러리 + 영역·연도·주제 선택 + 청취 진척
6. **학습 계획** (`/study-plan`) — D-day 카운트다운 + 페이스 토글(가벼움/표준/집중). 일정은 SRS 자동, 사용자는 페이스만 선택
7. **마이 페이지** (`/me`) — 프로필·시험일·뱃지·라이브러리

**Phase 2 추가 메뉴 2** [v3.0.1 — 자료 관리 폐기]:
- **오답 노트** (`/mistakes`) — 풀이의 again/hard 자동 합류 뷰
- **학습 분석** (`/study-analysis`) — Progress 변화·영역별 마스터율·활동 히트맵

**시스템 2**:
- **설정** (`/settings`)
- **로그아웃** (액션)

> 모바일 폰 화면은 본 헌법 개정 이후 별도 반응형 후속 작업으로 다룬다. 본 대회 기간 시연 디바이스는 태블릿 가로로 한정한다.
>
> v1.6 시절의 모바일 3탭(홈/학습/내 오답)은 v1.9 개정으로 폐지되었으며, v3.0에서 사이드바 구조가 임용 컨셉에 맞춰 재편성됨.

## 제13조의2 (학습 카드의 출처와 분류) [v3.0 전면 개정]

본 헌법 제13조의 사이드바 메뉴 중 카드 노출 위치는 다음 세 종류로 구분되며, 명칭의 혼용을 금한다. 코드의 타입·변수·UI 라벨은 본 표를 따른다. v2.x의 `VocabCard` / `StudyCard` 명명은 **폐지**된다.

| 종류 | 영문명 | 출처 | `cards.type` | 노출 위치 (v3.0) |
|---|---|---|---|---|
| 풀이 카드 | `QuizCard` | 시드 `exam_items` 중 **2014~2026 서술형만**. 본문 = **PDF 원본** (`stem_text` unpdf + `stem_image_path` PNG, v3.3 9항 정합) | `quiz` | 사이드바 "풀이" + 사이드바 "학습" 풀이 트랙 |
| 키워드 카드 | `KeywordCard` | 시드 `exam_items` 키워드 + **객관식 시대(2002~2013) 문항 분석에서 추출한 개념** (2002~2026 풀스코프, dedup). 본문은 **개념 정리 노트** 형식 (LLM 생성, `verified_answer=false`) — 정의·핵심 요소·출제 이력·관련 키워드 [v3.2 갱신] | `keyword` | 사이드바 "학습" 키워드 트랙 |
| 오답 카드 | `MistakeCard` | `QuizCard` 의 again/hard 자동 합류 + 사용자 직접 추가. 본문 = `QuizCard` 와 동일 (PDF 원본 참조) | `mistake` | 사이드바 "학습" 오답 트랙 (Phase 2: 별도 "오답 노트") |

1. 세 종류는 모두 **`ts-fsrs` SRS 큐**에 통합되어 복습 일정이 산출된다.
2. 단일 사용자 시점에서 카드 출처를 가시적으로 표시하여 신뢰성을 보장한다 (예: 출처 배지 — "2024 교직논술 1번", "키워드 #2022개정 교육과정", "오답 합류").
3. AI에 의한 *무작위 학습 카드 생성*은 어떠한 출처에서도 금지한다 (제14조 3항 정합).
4. 데시보드의 파생 위젯(취약 영역 분석 / 학습 성과 추이 / 추천 팟캐스트)은 위 세 종류 카드의 *집계·시각화* 산출물이며 새로운 카드 종류로 보지 아니한다.
5. **카드 간 합류** — 사용자가 `/study` 에서 `QuizCard` 를 풀다가 "again" 또는 "hard" 등급으로 평가하면 동일 문항이 `MistakeCard` 로 자동 합류된다. 합류 시 `source_item_id` 로 출처 추적이 가능하며, 동일 문항의 중복 합류는 차단된다.
6. 시드 카드는 모든 사용자가 동일한 본문을 보지만, **FSRS 상태와 등급 로그는 사용자별 분리** (`user_card_log`, `user_attempts`).
7. v3.0 시점에서 `vocab_cards` / `study_cards` 테이블은 마이그레이션으로 `cards` (단일 다형 테이블, type 분기)로 통합한다.
8. **[v3.2 신설] 객관식 시대(2002~2013) 문항의 처리** — 객관식 문항은 **풀이 카드(`type='quiz'`)로 출제하지 아니한다** (사용자 결정 2026-05-06). 객관식 문항 분석에서 추출한 핵심 개념·정의·관련 출제 이력은 `KeywordCard`의 본문(개념 정리 노트)에 흡수되어 사용자에게 학습 자료로 제공된다. 즉 객관식 시대 데이터는 (a) 기출분석 페이지의 트렌드 시각화(히트맵·토픽맵·로드맵) (b) 키워드 카드의 개념 정리 노트 본문 — 두 가지 경로로만 노출된다.

9. **[v3.3 신설] 시험 문제 본문 100% 정확성 보장 정책** — 사용자가 풀이 카드(QuizCard)에서 보는 **시험 문제 본문(stem)은 KICE 공식 PDF 원본**으로만 구성한다. 구체적으로:
   - **`stem_text`**: `unpdf` 라이브러리로 PDF 텍스트 레이어를 직접 추출 (벡터 PDF의 텍스트 객체를 OCR 없이 읽음 → 0% 오류). 검색·접근성 낭독·복사용.
   - **`stem_image_path`**: `pdftocairo`로 추출한 PDF 페이지 PNG. 사용자 화면에 보이는 본문 시각 자료 (표·도식·수식·빈칸 ㉠ 등 시험지 그대로).
   - **LLM은 본문을 생성·요약·재구성·교정하지 아니한다.** LLM의 stem 추론 결과는 어떤 형태로도 카드 본문에 노출되지 아니한다 (헌법 제30조 할루시네이션 차단 강화).
   - 운영자는 본문에 대해 "PDF와 동일한지" 외 변경 권한을 갖지 아니한다. 본문 정정이 필요하면 PDF 자체를 갱신한다.
   - 분석·태깅·답안·해설은 LLM 생성이며 별도 라벨(`verified_answer=false`)로 정직히 노출 (제18조의2 정합).
   - 본 정책의 정합 검증: schema의 `verified_text` 필드는 PDF 원본 직접 사용 시 **자동 true**, `verified_answer` 필드는 LLM 생성 시 **자동 false**로 시작하여 운영자 검수 시 true로 승격.

> **KeywordCard 본문 표준 구조 (v3.2 신설)**
> - **정의·요지**: 1~3문장 핵심 정의
> - **핵심 요소**: 2~5개 bullet
> - **관련 출제 이력**: 객관식·서술형 출제 좌표 + 묻는 방식 ("정의 묻기" / "적용 묻기" / "분석 묻기")
> - **함께 보는 키워드**: 의미·범주가 인접한 다른 키워드 2~5개

## 제13조의3 (팟캐스트 콘텐츠 — v3.0 신설)

1. 팟캐스트는 **NotebookLM 스타일 2인 화자 대화체**로, Gemini multi-speaker TTS로 자동 생성된다.
2. `podcast_episodes` 의 scope 는 다음 두 가지로 한정한다.
   - `shared` — 시드 영역×연도·주제 단위로 자동 생성, 모든 사용자에게 공유
   - `user` — 사용자가 자기 약점 영역·기출 자료를 선택해 즉석 생성, 본인만 청취
3. 모든 팟캐스트는 본문에 **"AI 생성, 공식 해설이 아님"** 안내를 포함하며, UI 에 `verified: false` 배지를 표기한다 (제3조의2·제18조의2 정합).
4. 청취 진척은 `podcast_progress` 테이블로 사용자별 재개 가능하게 보존한다.
5. 외부 팟캐스트(YouTube·기타 사설)의 임베드는 본 헌법 v3.0 시점에 도입하지 아니한다 (저작권·정직성 리스크).
